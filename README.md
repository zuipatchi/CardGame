# Card
マルチプレイヤーカードゲーム
ここには開発者向けのメモを記載する

## 開発の進め方
1. `/feature` を実行して新機能を実装する（ヒアリング→実装→テストまで自動で進む）
2. PlayMode: Window → General → Test Runner → PlayMode タブ → Run All で自動テストを実行する
3. Unity Editor で Play して動作確認する
4. 問題なければ `/ship` を実行してコミット・ドキュメント更新まで行う

決まったら記載する

## 機能一覧
- タイトル画面演出
  - カード軌道アニメーション（TitleCardSpherePresenter）：カードを1つの軌道リング上に配置し衛星軌道のように回転。Fisher-Yates シャッフルでランダム選出・1周ごとにラインナップ入れ替え。枚数・半径・傾き・速度を Inspector で調整可能
  - 背景画像（TitleBackgroundPresenter）：Addressables から背景をロードし、カメラ画角に合わせてSpriteRenderer で全画面表示。透明度（_alpha）を Inspector で調整可能
  - ロゴ表示（TitleLogoPresenter）：Addressables からロゴをロードし球の中心Z座標に配置。前半カードがロゴの手前・後半カードがロゴの奥に自然に重なるビルボード表示
  - スタートボタン（GameStartButtonPresenter）：画面全体タッチ対応・「PRESS START」ゴールドパルス点滅演出（USS transition + schedule）
  - Ambient Glows エフェクト（TitleAmbientGlowPresenter）：CFXR3 Ambient Glows を背景の上（SortingOrder=-50）・UI Toolkit の下にスポーン。EmissionRate 調整は ParticleSystem コンポーネントまたはスクリプトで multiplier 指定
- ホームシーン（Title → Home → DeckBuilder / Main / Matching の中継ハブ）
  - 「デッキ編集」「CPU対戦」「オンライン対戦」への遷移ボタン（カードパネル風デザイン・ホバーアニメーション・右寄せ配置）。コスト不足・コストオーバー時に対戦・マッチングボタンを押すとトーストを表示して遷移をブロック
  - Live2D キャラクター（Dog-kid）表示：画面左側に配置。登録した全モーションからランダムに再生→終了後に次をランダム選択のループ（AnimatorController 使用・FadeMotionList なし・CubismFadeController 無効化）
  - 食べ物スポーン・食事演出：画面左半分クリックで Food Live2D プレハブをスポーン（同時に最大1体・食べ終わるまで次はスポーン不可）。犬が歩いて近づき Eat アニメーションを同時再生、Food の Eat アニメーション終了後に削除（Dog-kid より前面表示）
  - 対戦結果連動天気演出：直前の対戦で負けた場合のみホーム遷移時に雨を表示し、表示後はリセット（次に負けるまで晴れ）。雨天時は vfx_Rain_01 パーティクルエフェクト（Y=15）と画面全体を暗くするオーバーレイ（rgba(0,0,0,0.4)）を表示。雨エフェクトは Live2D より前面に描画（Order in Layer: 19）
- BGMとSEの再生機能
- シーン管理機能（Commonシーンをベースに他のシーンを使用する）
  - フェードイン/アウトによる画面遷移演出
- オプション機能（BGM音量・SE音量・タイトルに戻る）
- オンラインマッチング機能（UGS Multiplayer Services）
  - クイックマッチ（空きルームへ自動参加 or 作成して待機・30秒タイムアウト）
  - ルーム作成して待機（120秒タイムアウト）
  - ルーム一覧から手動参加（満員ルーム・開始済みルームは非表示。クイックマッチ用ルームも非表示）
  - ホスト/クライアント間デッキ交換プロトコル（NetworkGameService）で初期手札を配布
  - サレンダー機能（オプションモーダル→確認ダイアログ→NGS_Surrender 通知。相手のサレンダー受信時は YOU WIN + 「対戦相手がサレンダーしました」表示）
- デッキ構築（Home → DeckBuilder → Home の遷移、ゲーム開始は Home から）
  - カード一覧をキャラ・スキル・イベントのセクション別に表示。カードをクリックで詳細モーダル表示・デッキパネルへドラッグ&ドロップまたは右クリックで追加（元カードはリストに残る）。コスト合計ルール（コスト超過でも追加自由・超過時は「コストオーバー」をデッキタイトル上に赤字表示）・背景画像（Addressables）・ゴミ箱ボタンで一括削除（確認ダイアログあり）。デッキパネルヘッダー右にコスト（X/50）と枚数（X枚）を横並び表示。デッキ一覧は【－】枚数【＋】ボタンで1枚単位の追加・削除が可能。デッキ一覧のカード名・イラストをクリックで詳細モーダル表示。各行左端の `≡` ハンドルをドラッグ&ドロップでカードの並び順を変更可能。「ID順」ボタンで C→S→E 順に自動ソート可能
  - デッキを PlayerPrefs に JSON 保存（カード追加・削除・一括削除・並び替え・ソートのたびに自動保存）・次回起動時に復元
- CPUデッキ構築シーン（CpuDeckBuilder）：Editor 専用・Build Settings 対象外。タイトル画面右下の「CPUデッキ編集」ボタン（Editor 時のみ表示）から遷移。プレイヤー用デッキ構築と同じレイアウト・機能（自動保存・分析モーダル・ゴミ箱ボタン・コストオーバー表示）。保存済みデッキは次回ゲーム開始時に CPU が使用（未保存時は AllCards フォールバック）
- カードゲーム盤面（Main シーン）
  - カードテンプレート（UXML）+ カードデータベース（ScriptableObject）、カードにイラスト（Sprite）設定可能
  - 盤面背景画像（Addressables）
  - 手札：初期5枚をシャッフル後に扇状表示（画面下端・60%スケール）、ホバーで1.5倍拡大＋最前面表示、ドロップ後に残りカードが詰まるアニメーション
  - 相手手札：画面上部に裏向き・反転表示（非インタラクティブ）
  - フィールド：自分フィールド・相手フィールドの2エリア、各最大5枚を中央寄せ表示、ドラッグ&ドロップで手札から配置
  - デッキ：自分（右下）・相手（左上・反転）それぞれ裏向きで積み重ね表示
  - カード裏返しアニメーション（DOTween）
  - ドロー演出：デッキ位置から手札へ飛翔→フリップ（自分・相手ともに初期5枚を0.12秒ずつずらして並走。相手は裏向きのまま）
  - カード情報表示：カード名・コストバッジを縦一列左寄せで表示。キャラカードは HP→攻撃力→（防御・素早さ横並びのL字配置）を追加表示、技カードは攻撃力のみ追加表示
  - 墓地ゾーン：プレイヤー墓地（左下）・相手墓地（右上）を常時表示。クリックで墓地詳細（タイトルに枚数表示）を開き、各カードをクリックで詳細モーダル表示
  - 攻撃矢印：フィールドカードをドラッグするとベジェ曲線の矢印を表示
  - OK/戻るボタン：画像ボタン（右下横並び）でカード配置・攻撃操作をステージング
  - ゲームロジック（ターンフロー）
    - マリガン（初期手札にキャラなし → 手札返却・リシャッフル・再ドロー。プレイヤー・CPU それぞれ1回）
    - 攻撃優先権（マリガン後にコイントスで決定。負けた側が1枚ドロー。素早さ同値のとき優先権保持者が先攻・行使後は相手に移る。保持者の墓地隣にコインアイコンを表示）
    - ターン管理：Draw（両者同時）→ キャラセット → 戦闘前1 → 戦闘前2（解決を含む） → 戦闘
    - キャラセットフェーズ（毎ターン Draw 直後に実行。両スロット埋まり時はスキップ。空スロットのみキャラカードを裏向きで配置可（パス可）。Resolve 後に新規配置のみ表向き＋コスト払い）
    - 戦闘前1：技カードを1枚だけ裏向きでフィールドにセット（キャラカード不可）
    - 戦闘前2（イベントフェーズ）：冒頭に Speed 比較で先攻後攻決定（「あなたが先です」/「相手が先です」表示。同値時は攻撃優先権保持者が先攻）→先攻から交互にイベントカード Ready 化。2連続パスで終了→同フェーズ末に Ready カードを逆順で処理（AtkBoost / DefBoost 効果を適用）
    - 戦闘フェーズ：全フィールドカードを表向き→先攻コスト払い（ATK=0はスキップ）→先攻ATK/後攻DEFカウントアップ→先攻突撃（ATK=0なら「NO ATTACK」）→先攻ダメージ・キャラ破壊→後攻コスト払い（ATK=0はスキップ）→後攻ATK/先攻DEFカウントアップ→後攻突撃（ATK=0なら「NO ATTACK」）→後攻ダメージ・キャラ破壊→技カードを墓地へ。ATK計算：キャラATK + 技ダメージ合計 + AtkBoost（スロット空ならATK=0）。先攻のキャラ破壊で後攻ATK=0に強制
    - イベントカード効果（AtkBoost: 次の戦闘フェーズの ATK を加算 / DefBoost: 次の戦闘フェーズの有効 DEF を加算 / Draw: 指定枚数ドロー / Negate: 解決フェーズで直後に処理されるカードの効果を無効化 / BanishChar: 相手キャラスロットのカードを墓地へ送る / Recover: 自分の墓地の上から指定枚数をデッキに戻してシャッフル / Switch: 自分のキャラスロットのカードを手札に戻した後、手札からキャラカードを1枚選んで表向きで配置してコストを払う。スロットが空なら効果失敗）。各カードに説明テキスト（`_description`）を設定可能

    - デッキ攻撃（ダメージ分だけ相手デッキを墓地へ）
    - 敗北条件（ドローフェーズ開始時・Draw エフェクト時・ダメージ計算後にデッキ0枚なら敗北。コスト支払い時はコスト > デッキ残枚数なら演出後に敗北・コスト == デッキ残枚数なら続行。上記以外のタイミングでのデッキ0枚到達は即死しない）
    - 勝敗演出（YOU WIN / YOU LOSE / DRAW をスケールアップで表示→タイトルに戻るボタン出現。YOU WIN 時は CFXR4 花火を7発・400ms 間隔で打ち上げ。RenderTexture＋加算ブレンド Canvas（Custom/FireworkAdditiveUI シェーダー）で UI の上に描画）
  - Resolve テキスト演出・攻撃アニメーション（DOTween）
  - フェーズ告知演出（ドローフェーズ 水色系 / キャラセットフェーズ・準備フェーズ・イベントフェーズ・FIGHT 黒白縁取り / イベントフェーズ冒頭に「あなたが先です」水色or「相手が先です」赤 / PAY X COSTS プレイヤー=水色・相手=赤）
  - カード確定時に「SET!」フラッシュ演出（プレイヤー=緑、CPU=赤）・演出終了後に次フェーズへ遷移
  - ターン開始ドロー（告知アニメーションと並走）
  - コスト警告（ステージング時にコスト支払いでデッキが0枚になる場合、OKボタン上部に赤字で警告表示）
  - カード詳細モーダル（手札・フィールド・キャラスロット・墓地モーダル内のカードをクリックで表示。左：カードイラスト（大）、右：種別・名前・ステータスアイコン＋ラベルを2カラム表示。キャラカード＝コスト・体力・攻撃力・防御・素早さ、スキルカード＝コスト・攻撃力、イベントカード＝コスト・説明テキスト。背景クリックで閉じる）
  - CPU 対戦：各フェーズで最適カードを自動選択
