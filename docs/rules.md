# ゲームルール

## デッキ構築ルール

### 枚数・コスト制限

| 項目 | 制限 |
|---|---|
| デッキ枚数 | ちょうど 30 枚（`DeckModel.MaxCards`）でのみ対戦可能 |
| コスト上限 | コスト合計が 80 以下（`DeckModel.MaxCost`）でのみ対戦可能 |
| 枚数オーバー | 30 枚を超えると「30枚を超えています」が赤字表示され、枚数カウンターも赤くなる |
| コストオーバー | コスト合計が 80 を超えると「コストが80を超えています」が赤字表示され、コストカウンターも赤くなる |
| Home での対戦ブロック | 枚数が 30 枚でない・コストが 80 超のいずれかでもトーストを表示して遷移をブロック |
| 同名カードの枚数制限 | なし（同じカードを何枚でも入れられる） |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（フィルタボタン横に「枚数 N/30」「コスト N/80」が常時表示）
3. 30 枚を超えると「30枚を超えています」、コスト合計が 80 を超えると「コストが80を超えています」が赤字で表示される
4. 「もどる」を押すと Home シーンに戻る（変更は自動保存済み）

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckSO`（ScriptableObject・Addressables キー `"Card/CpuDeck"`）から読み込む
  - カードIDリストが 1 件以上の場合：`CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - カードIDリストが空の場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）。編集内容は `CpuDeckSO`（`Assets/Data/CpuDeck.asset`）に保存され、ビルドに含まれる

## ゲーム開始時フロー（1回のみ）

```
マリガン選択（プレイヤー）
  ↓ 手札内容に関わらず YES/NO モーダルを表示
  ↓ YES → 「マリガン」告知 → 手札を全てデッキへ返却 → リシャッフル → 再ドロー
  ↓ NO → 手札をそのまま継続
  ↓ （1回限り）
マリガンチェック（CPU）
  ↓ 初期手札にキャラカードがなければ手札を全てデッキへ返却 → リシャッフル → 再ドロー（告知なし）
  ↓ （オンライン時は相手プレイヤーの決定を受信してアニメーション表示。下記参照）
  ↓
攻撃優先権コイントス
  ↓ ランダム（オンラインはホスト決定）でどちらが優先権を得るか決定
  ↓ コインフリップ演出 → 「優先権 獲得！」または「優先権 なし」を表示
  ↓ 優先権を得られなかったプレイヤーが手札を1枚ドロー
  ↓ 優先権保持者の墓地隣にコインアイコンを表示
  ↓
通常ターンループへ（1ターン目の Draw → キャラセット から開始）
```

**オンライン時のマリガン**: 両プレイヤーが独立して YES/NO モーダルで選択する。自分の選択後に `NGS_Mulligan` メッセージで決定を相手に送信し、相手の決定を受信するまで「対戦相手を待っています」を表示する。相手が YES を選択していた場合はアニメーション演出を表示する。マリガン要否の判定はホストが行わず、各プレイヤーが自律的に決定する。

## ターンフロー

```
ドローフェーズ
  ↓ 両プレイヤーが同時にデッキから1枚ドロー
  ↓
キャラセットフェーズ
  ↓ 各プレイヤーは1ターンに1枚だけキャラカードを裏向きでフィールドに出せる（パス可）
  ↓ 今ターンにすでにキャラを出したプレイヤーは自動パス
  ↓ Resolve → 新たに置かれたカードのみ表向き＋コスト払い
  ↓
戦闘前2フェーズ（イベントフェーズ）
  ↓ 戦闘フェーズの攻撃者として選択したキャラの Speed を比較して先攻後攻を決定（「あなたが先です」/「相手が先です」表示）
  ↓ Speed が高い方が先攻。同値の場合: 攻撃優先権保持者が先攻（行使後は相手に移る）
  ↓ 先攻プレイヤーから交互にイベントカードを手札から出す
  ↓ カードを出した直後に即座に1枚ずつ解決する（コスト払い → エフェクト → 墓地）
  ↓ 2連続パスで終了
  ↓
戦闘フェーズ
  ↓ 各プレイヤーがフィールドのキャラ1体と攻撃対象（相手キャラ or デッキ直撃）を選択
  ↓ 選択した攻撃者の Speed を比較して先攻後攻を決定
  ↓ 先攻側の処理 → 先攻ATKで指定対象へ攻撃 → ダメージ適用 → ダメージ ≥ HP なら相手キャラ破壊
  ↓ 後攻側の処理 → 後攻ATKで指定対象へ攻撃 → ダメージ適用 → ダメージ ≥ HP なら後攻キャラ破壊
  ↓
次のターンへ
```

## キャラセットフェーズの詳細

毎ターン、ドローフェーズの直後に実行される。各プレイヤーは1ターンに1枚まで、キャラカードを裏向きで自フィールドに出せる（フィールドには何枚でも出せる）。

1. **1ターン1枚制限**: `GameModel.CanPlayerSetChar` / `CanOpponentSetChar` フラグで管理。今ターンにすでにキャラをセットしたプレイヤーは自動パス（`EndTurn` でリセット）
2. **まだセットしていないプレイヤーが行動**
   - 手札からキャラカードを1枚裏向きでフィールドに置く（パス可）
   - CPU戦: `UniTask.WhenAll` で自分と CPU の選択を並列実行
   - オンライン: 相手カード受信タスクを並列起動し、自分の入力・送信後に相手受信完了を await
3. 両者の配置・パスが完了したら Resolve アニメーション
4. Resolve 後、**新たに置かれたカードのみ**表向きにしてコストを払う

---

## 戦闘前2フェーズ（イベントフェーズ）の詳細

1. **先攻後攻決定**: 戦闘フェーズで攻撃者として選択したキャラの Speed を比較する（攻撃者なし = Speed 0 として扱う）
   - Speed が高い方が先攻 → 「あなたが先です」（水色）/ 「相手が先です」（赤）を表示
   - Speed が同値の場合: **攻撃優先権**を持つプレイヤーが先攻。行使後は優先権が相手に移る（ただし両者の ATK が 0 の場合は優先権を消費しない）
   - 優先権はゲーム開始時のコイントスで決定（オンラインはホストが決定した `OnlineInitialState.IsLocalFirst` を使用）
2. 先攻プレイヤーがイベントカードを手札から1枚出す（スキップ可）
3. カードを出した直後にコストを払い、効果を即座に解決して墓地へ送る
4. 相手プレイヤーがイベントカードを手札から1枚出す（スキップ可）→ 同様に即時解決
5. 交互にカードを出し続け、**2連続パス**で戦闘前2フェーズ終了

### イベント効果一覧

| EventType | 効果 | EventValue の意味 |
|---|---|---|
| AtkBoost | 次の戦闘フェーズの ATK に加算 | 加算値 |
| DefBoost | 次の戦闘フェーズの有効 DEF に加算 | 加算値 |
| Draw | 発動プレイヤーが指定枚数デッキ上から手札に加える。1枚引くごとにデッキが 0 になるか確認し、0 になった時点でゲームオーバー判定・残りのドローを中断 | ドロー枚数 |
| BanishChar | 相手フィールドの先頭キャラ（`Characters[0]`）を墓地へ送る。相手フィールドにキャラがいない場合は空振り | 使用しない（0 固定） |
| Recover | 発動プレイヤー自身の墓地の**上から**指定枚数を取り出し、自デッキに加えてシャッフルする。墓地枚数が指定枚数未満の場合は存在する全枚数を回収する | 回収枚数 |
| Switch | 発動プレイヤー自身のフィールドのキャラを1体選んで（複数いる場合はクリックで選択）手札に戻し、手札からキャラカードを1枚コストを払って配置する。フィールドにキャラがいない場合は効果なし | 使用しない（0 固定） |
| CharDamage | 相手キャラに直接ダメージを与える。ダメージ = EventValue − 相手キャラ DEF − DefBoost（0 未満は 0）。damage ≥ 相手キャラ HP でキャラ破壊＋デッキダメージ、0 < damage < HP でデッキダメージのみ、damage = 0 で NO DAMAGE。相手キャラなしの場合は EventValue がそのままデッキダメージ。HP は累積せず各判定は独立 | ベース ATK 値 |
| Evolve | 自分のフィールドのキャラを1体選んで（複数いる場合はクリックで選択）墓地に送り、手札から犠牲キャラより高コストのキャラカードを1枚コストなしで配置する。フィールドにキャラがいないまたは手札に適格カードがない場合は何もしない。配置時にエフェクト再生。解決後に速さを再評価して戦闘フェーズの先攻後攻を決定 | 使用しない（0 固定） |
| Poison | 同ターンの戦闘フェーズで相手キャラにダメージ > 0 が入った場合、戦闘終了後に相手キャラを破壊する。ダメージが 0 の場合（DEF で全防御）は発動しない。相手キャラが戦闘で既に破壊されていれば追加破壊はなし。`_playerPoisoned` / `_opponentPoisoned` フラグで管理し、`ResetBoosts` でターン終了時にリセット | 使用しない（0 固定） |
| DeckMill | お互いのデッキ上から EventValue 枚を墓地へ送る。**発動プレイヤー自身のデッキを先に**処理し、次に相手デッキを処理する（相手が発動した場合は相手デッキ先 → 自デッキ後）。要求枚数を出し切れなかった場合に敗北が確定する（先に処理したデッキが不足 → そのデッキのオーナーが敗北、後に処理したデッキが不足 → そのデッキのオーナーが敗北）。デッキ枚数がちょうど EventValue の場合は全枚ミルしても敗北にならない。ミルしたカードに `TriggerOnGrave` がある場合は墓地トリガーが発動する | ミル枚数 |
| BattleEndMill | **永続効果**。カードを発動したプレイヤーの相手デッキ上から、毎ターン戦闘フェーズ終了後に EventValue 枚を墓地へ送る。発動時と毎ターンのトリガー時に毒パーティクルで演出。両プレイヤーが BattleEndMill を持つ場合は発動プレイヤー側の効果から処理する（相手デッキ → 自デッキの順）。要求枚数を出し切れなかった場合に敗北確定（デッキ残枚数 < ミル枚数の場合。ちょうどミル枚数の場合は敗北しない）。`_localBattleEndMillValue` / `_opponentBattleEndMillValue` フィールドで永続管理し、`ResetBoosts` ではリセットしない。**同一プレイヤーが複数枚発動した場合、後から発動したカードの EventValue で上書きされる（累積しない）** | ミル枚数（毎ターン） |

### グレイブトリガー（EventCardData.TriggerOnGrave）

`EventCardData` に `TriggerOnGrave: bool` フラグを追加した特殊な効果。

- `TriggerOnGrave = true` のイベントカードは、そのカード自身がデッキから墓地に送られた瞬間に効果が発動する
- 発動タイミング：コスト支払い・戦闘ダメージの両方で `PlayDeckDamageAsync` 経由で送られたとき
- 発動演出：カードのオーナー側の墓地近くに「TRAP！」フローティングラベルを表示してから効果を適用
- 効果は `EventType` と `EventValue` で定義し、通常のイベント効果（CharDamage / AtkBoost / Draw / Recover など）をそのまま使用できる
- 同じ `PlayDeckDamageAsync` の中で複数の TriggerOnGrave カードが送られた場合は、送られた順にトリガーを処理する
- カードのオーナー（`CardView.IsOpponent`）からトリガー発動者の `isLocal` を決定する

## 戦闘フェーズの詳細

0. **攻撃選択**：各プレイヤーが自フィールドのキャラ1体と攻撃対象を選択する
   - プレイヤー: キャラから矢印をドラッグして相手キャラに向ける（target = 相手キャラ）、または相手キャラがいない場合は自動的に最高ATKキャラでデッキ直撃（target = null）
   - フィールドにキャラがいない場合は自動パス（攻撃者なし）
   - パスボタンでも攻撃をスキップ可能
   - CPU: `CpuAgent.ChooseBattleAttack` で最高ATKキャラを選択、最低ATK相手キャラを対象にする
1. **先攻フィールドカードのみ**を表向きにする（後攻フィールドカードは後攻フェーズ開始直前に表向きにする）
2. 素早さ同値で攻撃優先権が行使された場合：「優先権 行使！」アナウンスと同時にコインが画面中央へ飛翔し、アナウンス消去後に相手の墓地隣へ移動して優先権が移る。**両者の ATK が 0（双方 NO ATTACK）の場合は優先権の消費・移譲は行われない**
3. **先攻フェーズ**：ATK > 0 のとき「PAY X COSTS」アナウンス → コスト払い（ATK = 0 のときコスト払いはスキップ）→ 先攻 ATK/後攻 DEF カウントアップ演出 → 先攻 ATKカウンターアイコンが指定対象（相手キャラまたは相手デッキ）へ突撃（ATK = 0 なら突撃せず「NO ATTACK」ラベル表示）→ ATK > 0 かつダメージ = 0 なら防御側に「NO DAMAGE」水色ラベルを表示して 0.25 秒待機 → ダメージ > 0 ならダメージ数字アイコンとキャラ破壊飛翔を同時に実行（ダメージアイコン出現＋ホールド後に両者が同速で飛翔し同着） → 先攻デッキダメージ適用（ダメージ量が相手デッキ残枚数を超えた場合は先攻勝利終了）→ ダメージ ≥ キャラ HP なら相手キャラ破壊（墓地飛翔）
4. **後攻フェーズ**：ATK > 0 のとき「PAY X COSTS」アナウンス → コスト払い（ATK = 0 のときコスト払いはスキップ）→ 後攻 ATK/先攻 DEF カウントアップ演出 → 同様に突撃・ダメージ飛翔 → ATK > 0 かつダメージ = 0 なら「NO DAMAGE」水色ラベルを表示 → デッキダメージ適用（ダメージ量が先攻デッキ残枚数を超えた場合は後攻勝利終了）→ ダメージ ≥ キャラ HP なら先攻キャラ破壊

### ATK の計算

- ATK = **キャラATK + AtkBoost**
- 以下の場合は ATK = 0（突撃モーションなし・「NO ATTACK」グレーラベルを表示）:
  - 攻撃者を選択しなかった（フィールドにキャラなし、またはパス）

### ダメージ計算

- ダメージ = `MAX(0, ATK - 有効DEF)`。有効DEF = DEF + DefBoost
- ダメージ枚数分だけ相手デッキの上からカードを墓地へ送る
- ATK が有効DEF以下の場合はダメージなし（0枚）
- 攻撃対象がデッキ直撃（target = null）の場合、有効DEF = 0（DefBoost も無効）として計算する

### キャラ破壊

- デッキダメージ適用後、**ダメージ ≥ キャラの HP** の場合、そのキャラカードを墓地へ送る
- デッキダメージとキャラ破壊は独立して発生する（どちらも同じターンに起こりうる）
- キャラが破壊されても次ターン以降にキャラセットで補充可能

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コストの枚数分だけ**自分のデッキ上から墓地**へカードを送る。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（キャラセット） | Resolve 後、カードが表向きになった直後 |
| イベントカード（戦闘前2） | OK ボタンで Ready を確定した直後 |

- コスト 0 のカードは墓地送りアニメーションをスキップする
- コスト > デッキ残枚数の場合は残り全枚数を払い切り、演出完了後に敗北判定
- コスト == デッキ残枚数の場合はデッキが 0 枚になるが敗北しない

## カード状態

| 状態 | 説明 |
|---|---|
| Normal | フィールド上で待機中 |
| Ready | 解決フェーズで処理される待機状態。対象はすでに決定済み（視覚的な区別なし） |
| Resolve | 解決フェーズで現在処理中のカード。黄色枠で表示 |

## 勝敗条件

以下のいずれかが発生した時点で敗北。

| 発生タイミング | 条件 |
|---|---|
| ドローフェーズ開始時 | 自分のデッキが 0 枚 |
| コスト支払い時（キャラセット・戦闘前2・戦闘フェーズ） | 払うコストがデッキ残枚数より多い（コスト > デッキ残枚数）。演出で残り全枚数を払い切った後に敗北。コスト == デッキ残枚数の場合は敗北しない |
| Draw エフェクトで次の 1 枚を引こうとした時 | 自分のデッキが 0 枚 |
| 戦闘フェーズ・ダメージ計算後 | ダメージ量が自分のデッキ残枚数を超えた（要求枚数を出し切れなかった場合）。ちょうどダメージ量と残枚数が同じ場合は敗北しない |
| CharDamage イベント効果後 | ダメージ量が自分のデッキ残枚数を超えた（要求枚数を出し切れなかった場合） |
| DeckMill イベント効果後 | お互いのデッキで要求枚数を出し切れなかった側が敗北（発動プレイヤーのデッキを先処理） |
| BattleEndMill 毎ターン処理後 | 要求枚数を出し切れなかった側が敗北 |

- 上記以外のタイミングでデッキが 0 枚になること自体は即座に敗北にならない
- 勝敗決定時に画面中央に「YOU WIN」「YOU LOSE」「DRAW」の演出を表示し、ホームに戻るボタンが出現する
