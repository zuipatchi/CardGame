# ゲームルール

## デッキ構築ルール

### 枚数制限

| 項目 | 制限 |
|---|---|
| デッキ枚数 | ちょうど 30 枚（`DeckModel.MaxCards`）でのみ対戦可能 |
| 枚数オーバー | 30 枚を超えると「30枚を超えています」が赤字表示され、枚数カウンターも赤くなる |
| Home での対戦ブロック | 枚数が 30 枚でないときにトーストを表示して遷移をブロック（枚数制限 OFF 時は 1 枚以上であれば許可。空のときは「デッキが空です」トースト） |
| 同名カードの枚数制限 | 同一カード（ID 基準）は 3 枚まで（`DeckRuleModel.MaxCopiesPerId`）。4 枚目を追加しようとすると「同名カードは3枚までです」トーストを表示して拒否（ドラッグ／右クリック／＋ボタンすべて）。保存済みデッキの読み込みは制限対象外 |
| 制限の ON/OFF | Editor 再生時のみ Home シーン左上のトグルで切り替え可能。「同名3枚制限」（`DeckRuleModel.LimitSameCards`）と「デッキ枚数制限」（`DeckRuleModel.LimitDeckCount`）の 2 つがあり、後者を OFF にすると 30 枚ちょうどでなくても（1 枚以上なら）対戦を開始できる。各トグルの状態は `EditorPrefs` に保存され、再生をまたいで前回の状態を引き継ぐ。ビルドにはトグルが存在せず `EditorPrefs` も読まないため常に ON。CPU デッキ構築は Home トグルに依存せず常に ON |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（フィルタボタン横に「枚数 N/30」が常時表示。コスト表示はなし。同一カードは 3 枚まで）
3. 30 枚を超えると「30枚を超えています」が赤字で表示される
4. 「もどる」を押すと Home シーンに戻る（変更は自動保存済み）

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckSO`（ScriptableObject・Addressables キー `"Card/CpuDeck"`）から読み込む
  - カードIDリストが 1 件以上の場合：`CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - カードIDリストが空の場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）。編集内容は `CpuDeckSO`（`Assets/Data/CpuDeck.asset`）に保存され、ビルドに含まれる

## ゲーム開始時フロー（1回のみ）

```
先攻後攻コイントス
  ↓ ランダム（オンラインはホスト決定）で先攻・後攻を決定
  ↓ コインフリップ演出 → 「先攻！」または「後攻！」を表示
  ↓ （配牌の前に行う。手札枚数が先攻3枚・後攻5枚で変わるため）
  ↓
配牌（先攻3枚・後攻5枚）
  ↓ 先攻プレイヤーは3枚、後攻プレイヤーは5枚を初期手札として引く（先攻有利の補正）
  ↓
マリガンチェック（プレイヤー）
  ↓ 手札内容に関わらず YES/NO モーダルを表示（確認中も手札カードをクリックしてカード詳細を閲覧可能）
  ↓ YES → 「マリガン」告知 → 手札を全てデッキへ返却 → リシャッフル → 初期手札枚数（先攻3/後攻5）を再ドロー
  ↓ NO → 手札をそのまま継続
  ↓ （1回限り）
マリガンチェック（CPU）
  ↓ 初期手札にキャラカードがなければ手札を全てデッキへ返却 → リシャッフル → 再ドロー（告知なし）
  ↓ （オンライン時は相手プレイヤーの決定を受信してアニメーション表示。下記参照）
  ↓
通常ターンループへ（先攻プレイヤーのドローフェーズから開始）
```

- 初期手札枚数は `MulliganRule`（先攻=3枚 `FirstPlayerHandSize`／後攻=5枚 `SecondPlayerHandSize`）で定義する。オフラインはプレイヤー・CPU それぞれの先攻/後攻に応じて配り、オンラインはホストが双方の枚数を配る。

**オンライン時のマリガン**: 両プレイヤーが独立して YES/NO モーダルで選択する。自分の選択後に `NGS_Mulligan` メッセージで決定を相手に送信し、相手の決定を受信するまで「対戦相手を待っています」を表示する。相手が YES を選択していた場合はアニメーション演出を表示する。マリガン要否の判定はホストが行わず、各プレイヤーが自律的に決定する。

## ターンフロー

ターンは先攻プレイヤーから交互に繰り返す。

```
【アクティブプレイヤーのターン開始】
ターン開始時効果（ドロー前）
  ↓ 場のキャラの OnTurnStart 効果を並び順に発動
  ↓ プレイ済みの永続イベント（EventCardTrigger.OnTurnStart）を墓地からせり出して発動
  ↓
ドローフェーズ
  ↓ アクティブプレイヤーのみデッキ上から3枚ドロー（デッキが 0 枚の場合はスキップ）
  ↓ ただし両プレイヤーの初手（各自がこのゲームで最初に迎える自分のドローフェーズ）はドローなし（先攻有利の補正）
  ↓ 判定はプレイヤーごとの初手フラグで行い、ExtraTurn で通算ターン番号がずれても各自の初手のみ0枚になる
  ↓ ドロー枚数は効果で増減する: EventType.DrawSkipNext は予約された側のドローを 0 枚に、EventType.DrawNextTurnStart は予約枚数を上乗せ（いずれも消費。詳細は event.md）
  ↓ 手札が上限（8枚）の状態で引いたカードは手札に加えず墓地へ送る（バーン）。デッキからは引かれる（自分のドロー時のみ「手札が上限 → 墓地へ」トースト表示）
  ↓ オンライン時: アクティブプレイヤーが NGS_Draw を送信、相手がアニメーション再生（0枚でも同期のため送信）
  ↓
メインフェーズ
  ↓ アクティブプレイヤーは EndButton を押すまで、枚数・攻撃回数の制限なく以下を繰り返せる
  ↓   キャラを出す: 手札のキャラカードをフィールドに表向きで配置 → 手札からコスト枚数選択して墓地へ
  ↓   イベントを使う: 手札のイベントカードをフィールドへ → 即時解決（手札コスト選択 → 効果 → 墓地）
  ↓   攻撃する: フィールドのキャラから矢印をドラッグして相手キャラへ攻撃（各キャラ1回まで）
  ↓ EndButton（画面右）でターン終了
  ↓
ターン終了 → 相手のターンへ
```

- 例外: `EventType.ExtraTurn` の効果が発動したターンは、相手にターンを渡さず**同じプレイヤーがもう一度ターンを行う**（ターン開始時効果→ドロー→メインフェーズ）。詳細は [event.md](event.md) を参照。

## メインフェーズの詳細

アクティブプレイヤーは **EndButton を押すまで、枚数・攻撃回数の制限なく**以下のアクションを繰り返せる。各アクション後も同じメインフェーズが続く。

### アクション選択肢

| アクション | 操作方法 | 処理 |
|---|---|---|
| キャラを出す | 手札のキャラカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドに表向きで配置 → 手札からコスト枚数を選択して墓地へ送る（フィールドのキャラは9体まで。満杯時はドロップ拒否） |
| イベントを使う | 手札のイベントカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドへ配置 → 即時解決（手札コスト選択 → 効果 → 墓地）|
| 攻撃する | フィールドのキャラから矢印をドラッグして相手キャラへ | 攻撃実行（下記「攻撃の詳細」参照）|
| ターン終了 | EndButton（画面右） | 自ターンを終了して相手のターンへ |

- コスト 0 のカードはドロップ後すぐ OK ボタンが有効になる
- コスト選択中は EndButton（ターン終了）を非表示にし、戻る（Back）ボタンを EndButton があった右端に、OK ボタンをその左に配置する
- Back ボタンでコスト選択をキャンセルしてカードを手札に戻せる
- ステージ中（カードを場に仮置き・コスト選択中を含む）に**別の手札カードをフィールドへドラッグ**すると、仮置き中のカードを手札へ戻して新しいカードに差し替えられる（Back ボタンの代わりの操作）。戻したカードは手札データへ同期登録されるため、差し替え後に出すカードのコスト計算でも支払い要員として数えられる。フィールド外へのドロップでは差し替わらず元のステージを維持する
- AutoOk が有効な場合、コスト 0 はドロップ直後に自動で OK、コストありは必要枚数を選択した時点で自動で OK

### 手番の制限時間

- 自分のメインフェーズには **60秒の制限時間**がある（`TurnTimeLimitSeconds`）。残り時間は左下のターン表示の上にカウントダウン表示する（`TurnTimerView`。自分のメインフェーズ中のみ表示）
- 残り **10秒以下**（`TurnTimeWarningSeconds`）で数字が赤く拡大し、毎秒の警告 SE が鳴る
- 0 になると「時間切れ！」を告知して**自動的にターン終了（パス）**する
- 時間切れの時点でカードを出しかけ（ステージ中）・コスト選択中の場合は、`OnBackClicked` と同じ処理でカードを手札へ戻し、コスト選択をキャンセルしてからパスする
- カード効果の**解決アニメーション中**に時間切れになっても演出は中断せず、演出完了後（次の入力待ち）にパスする（オンラインの同期ずれを防ぐため）。実装は `_mainActionTcs` を Pass で完了させる方式で、トークンのキャンセルは行わない
- タイマーは**自分の手番のみ**稼働する。CPU・オンライン相手の手番は各クライアントが自分の手番で同じ自動パスを行うため、相手が放置しても相手側で時間切れ→パスが送られてくる

### 攻撃回数と召喚酔い

- **各キャラの攻撃はターンに1回まで**（`_attackedThisTurn` で管理。攻撃済みキャラは矢印を出せない）
- **召喚酔い**: このターンに登場したキャラ（配置・召喚・Switch / Evolve での配置）はそのターン攻撃できない。次の自ターンから攻撃可能。判定は自メインフェーズ開始時のフィールド面子のスナップショット（`_playerSeasonedChars` / `_opponentSeasonedChars`）で行う
- **速攻**: `CharacterCardData.Haste`（SO のチェックボックス）を持つキャラは召喚酔いしない＝場に出したターンから攻撃できる（`CanCharAttack` で seasoned 判定をバイパス。1ターン1回の攻撃制限は維持）。速攻持ちはカードと詳細モーダルに SpeedIcon を表示
- このターン攻撃できる自キャラはシアン色でハイライト表示される（USS `attackable-char`）
- 攻撃できる自キャラが1体以上いる場合、攻撃可能な相手キャラを赤〜オレンジ系でハイライト表示する（USS `attack-target-char`、攻撃側のシアンと区別）。「いずれかの攻撃可能な自キャラが攻撃できる対象」を守護・飛行を考慮して強調する（例: 守護がいる場合は守護持ちのみだが、飛行を持つ自キャラがいればそれが狙える対象も加わる）。メインフェーズのループで毎回再計算される
- オンラインでは EndButton 押下を Pass アクションとして相手へ送り、ターン終了の合図に使う（追加メッセージは不要）

### タップ

カードを 90° 横向きに倒した状態を**タップ**と呼ぶ（`CardView.IsTapped`。`CardState` とは別の実行時フラグで、見た目は内側 `_cardRoot` の回転で表現する。手札の扇・フィールド配置で使う外側 `style.rotate` とは独立）。タップ中は `game-card--tapped` クラスが付き、ネームラベルとアイコンを `-90°` 逆回転して読める向きに保ち、ネームを横向きカードの上辺・アイコンを底辺の横並びに再配置する（USS `Card.uss`）。

- **攻撃したらタップ**: キャラ攻撃・デッキ攻撃のいずれも、攻撃した攻撃側キャラはタップする（ATK 0 の盾ブロックでもタップ）。突進演出の後に横向きへ倒れる（`TapAttackerAsync`）
- **タップ状態のキャラにしか攻撃できない**: 相手キャラへの攻撃は、対象が**タップ済み**のときのみ可能。未タップ（＝未行動）のキャラは攻撃対象にならず、攻撃ハイライト（`attack-target-char`）もされない。未タップキャラへ矢印を引くと「タップ状態のキャラにしか攻撃できません」トースト。判定は守護・飛行・防人の対象制限に「タップ済み」要件を上乗せして `CanAttackChar` で中央化する
- **デッキ攻撃はタップ制限の対象外**: 相手デッキへの直接攻撃（ミル）は相手キャラのタップ状況に関係なく可能（守護・飛行・防人の制限のみ `CanAttackDeck`）。ただし攻撃側はタップする。これにより、相手キャラが全員アンタップで攻撃対象がいない序盤でも、デッキを攻めることで自分が寝て相手に的を与える、というテンポの駆け引きが生まれる
- **アンタップ**: 自分のターン開始時（メインフェーズ開始時）に、自分の場の全キャラをアンタップ（縦に戻す）する（`UntapField`。即時・演出なし）
- **守護・防人の自動タップ**: 守護または防人を持つキャラは、**自分のターン終了時に毎ターン自動でタップ**する（`AutoTapGuardiansAndSakimoriAsync`。既にタップ済みなら何もしない）。アンタップは自分のターン開始時のため、相手ターン中は常にタップ済み＝常に攻撃対象になる（守護・防人の壁としての役割と噛み合う）
- カード効果（`DamageEnemy` 等）の対象選択はタップ状況の影響を受けない（守護・飛行と同じく**攻撃のみ**に作用する）
- バウンス／スイッチで手札に戻ったキャラはアンタップ状態にリセットされる（`CardView.ResetRuntimeState`）
- **召喚酔い**・**1ターン1回の攻撃制限**はタップとは別の軸で、従来どおり維持する（タップは「攻撃される側になるか」、召喚酔い・攻撃済みは「攻撃できるか」を制御する）
- オンライン対戦ではタップ/アンタップは両クライアントが同じ行動・ターン構造を再生して決定的に一致するため、追加のネットワークメッセージは不要

### CPU のアクション選択順

毎ループ以下を判定し、いずれも該当しなくなるまで繰り返す（該当なしでターン終了）。攻撃の候補は「このターン未攻撃かつ召喚酔いしていないキャラ」のみ。

1. **lethal デッキ攻撃**: デッキを直接攻撃でき（`CanAttackDeck`）、ATK がプレイヤーのデッキ枚数を**超える**攻撃者がいれば、デッキ攻撃で空デッキからさらにミルさせて（オーバーリミット）勝利を狙う（最高ATKを選ぶ）
2. **キャラ攻撃**: 攻撃可能なキャラがいて、かつ相手フィールドに合法な攻撃対象がいれば攻撃（`CanAttackChar` で守護・飛行を考慮し、合法な対象を持つ攻撃者の中で最高ATK→対象は最低ATKを選ぶ）
3. **チップミル（デッキ攻撃）**: 合法なキャラ攻撃対象がいない場合、デッキを直接攻撃できる攻撃者がいればデッキを削る（最高ATKを選ぶ）
4. 手札に**コストを支払えるキャラカード**があれば配置
5. 手札に**コストを支払えるイベントカード**があれば使用
6. 上記いずれもなければターン終了

> 配置・使用の対象は「自身を除いた手札の支払い可能量（`CostPaymentValue` 合計）がコスト以上」のカードに限る（`MainPresenter.CpuCanAffordCost`）。ローカルプレイヤーと同じくコストの踏み倒しはできない。`NextCardCostFree` 武装時とコスト 0 のカードは常に支払い可能。

### 攻撃の詳細

- **一方向攻撃のみ**（カウンター攻撃なし）
- 攻撃対象: 相手フィールドの表向きキャラ、または**相手デッキ**（矢印で選択）
- キャラ攻撃: 相手にキャラがいない場合はキャラへの攻撃は不可
- **デッキへの直接攻撃（ミル）**: フィールドのキャラから矢印を**相手デッキ**へドラッグすると、攻撃キャラの **ATK 枚**だけ相手デッキの上から墓地へ送る（デッキ枚数が ATK 未満なら全枚数）。ATK 0 はシールドアイコン表示で不発。各キャラ1回までの攻撃制限・召喚酔いはキャラ攻撃と共通（デッキ攻撃もそのキャラのターンの攻撃を消費する）。**オーバーリミット**により、相手デッキを0枚にする（ATK == 残り枚数）だけでは敗北せず、**さらにミルしようとした瞬間**＝ ATK が相手デッキ枚数を**超える**ときに相手はデッキ切れで敗北する（「勝敗条件」参照）。**相手デッキが0枚（オーバーリミット状態）でもデッキ攻撃で finish できる**：空デッキへの ATK 1 以上のデッキ攻撃は、最初のミルで即敗北を誘発する（ATK 0 はシールドで不発）。守護・防人で対象が強制される間は空デッキも狙えない。攻撃できる自キャラがいる間は相手デッキを赤〜オレンジ系でハイライト（USS `deck-view--attack-target`。0枚でもハイライトする）
- **ダメージトリガー（デッキダメージのカウンター）**: ミルで墓地へ送られるカードが**ダメージトリガー**（`CardData.TriggerOnGrave`）を持つ場合、墓地行きの代わりに**デッキの持ち主**がコストを支払わずにそのカードを使用する（`ResolveGraveTriggerFromDeckAsync`）。発動時は**デッキから表向きのカードをせり上げて「DT」ラベルを表示**するカットイン演出を挟み（`PlayGraveTriggerCutInAsync`）、その後**移動先で閃光**を再生する（`PlayGraveTriggerBurstAsync`。連続トリガーでも閃光は await して1個ずつ）。キャラは**持ち主の場の空きスロットへ飛翔・着地してから実カードへ差し替わる**形で召喚され（`SummonGraveTriggerCharAsync`。登場時効果 OnEnter も発動し、そのまま場に残る。着地の閃光と重複する召喚の登場ポップは省略）、イベントは**場中央へせり出して**効果を解決してから墓地へ送られる。**デッキから墓地へ送られた場合のみ**発動し、戦闘破壊・コスト支払い・手札バーンなどデッキ以外の経路では発動しない（[event.md](event.md)「ダメージトリガー」）
- **守護（タウント）**: 相手フィールドに**守護**を持つキャラがいる間は、守護持ちキャラにしか攻撃できない（守護以外のキャラ・**相手デッキ**への攻撃は不可）。守護が複数いればそのいずれか。守護が破壊されれば通常の対象選択に戻る。守護持ちキャラはカードの HP アイコンの下に ShieldIcon を表示。守護は `CharacterCardData.Guardian`（SO のチェックボックス）で設定（攻撃のみを制限し、`DamageEnemy` 等のカード効果の対象選択には影響しない）
- **飛行**: `CharacterCardData.Flying`（SO のチェックボックス）を持つキャラは、**守護を無視**して攻撃対象（相手キャラ・相手デッキ）を選べ、かつ**飛行か防人を持つキャラからしか攻撃されない**（飛行・防人なしキャラは飛行キャラを攻撃不可）。飛行同士は攻撃可能。守護かつ飛行を持つキャラは、飛行でしか攻撃できず、かつ守護として攻撃を強制する（飛行なし攻撃者は対象がなくなり実質攻撃不可）。攻撃のみに作用し、`DamageEnemy` 等のカード効果の対象選択には影響しない。判定はキャラ攻撃が `CanAttackChar`・デッキ攻撃が `CanAttackDeck` に中央化し、プレイヤー攻撃・CPU・攻撃ハイライトで共用する。飛行持ちはカードと詳細モーダルに FlyIcon を表示
- **防人（さきもり）**: `CharacterCardData.Sakimori`（SO のチェックボックス）を持つキャラは、**守護を兼ねる対空ガード**として働く。① **飛行を持つ攻撃者**は、相手フィールドに防人がいる間は**防人を優先して攻撃しなければならない**（守護は無視できるが防人は無視できない。防人以外のキャラ・相手デッキへの攻撃は不可）。② **飛行を持たない攻撃者**は、相手フィールドに守護か防人がいる間は**守護か防人のいずれかを優先して攻撃しなければならない**（防人は守護と同じく地上の壁にもなる）。③ 防人を持つキャラ自身は**飛行を持つキャラに攻撃できる**（通常「飛行は飛行からしか攻撃されない」の例外）。守護と防人が両方いる場合、飛行なし攻撃者は守護か防人のどちらか、飛行攻撃者は防人を狙う。攻撃のみに作用し、カード効果の対象選択には影響しない。防人持ちはカードと詳細モーダルに ProtectorIcon を表示
- **キーワード能力の実行時付与**: 守護・速攻・飛行・防人の4能力は SO のチェックボックスに加え、**`GrantKeyword` 効果でゲーム中に味方キャラへ永続付与できる**（[event.md](event.md)）。付与した能力も SO 固有の能力と同様に攻撃ルールへ反映される。攻撃ルールの判定は SO 固有フラグを直接読まず、付与分を含めた実行時状態（`CardView.HasGuardian` / `HasHaste` / `HasFlying` / `HasSakimori` ＝ SO 固有 OR 付与）を参照する。盤面のアイコンには付与が反映されるが、カード詳細モーダルは攻撃力/HP バフ同様カード固有能力のみ表示する

### ダメージ計算

- ダメージ = `ATK`（防御力なし）
- ダメージ > 0: 攻撃対象キャラの現在 HP からダメージ値を減算する（累積ダメージ）
- ダメージ = 0: シールドアイコンをフローティング表示して攻撃を終了

### キャラ破壊

- 攻撃を受けたキャラの現在 HP からダメージ値を減算する（累積ダメージ）
- HP が **0 以下**になったキャラカードを墓地へ送る
- HP の残量は `CardView.CurrentHp` で管理し、カード上の HP ラベルにリアルタイムで反映される
- キャラをフィールドに配置するたびに HP は最大値（`CardData.Hp`）にリセットされる（Switch で手札に戻して再出しした場合も同様）
- キャラが手札に戻る（Switch / Bounce / BounceAll）と、場で受けた実行時の変化（攻撃力・HPバフ、被ダメージ、付与キーワード）はすべて初期状態にリセットされる。リセットは手札へ戻す共通処理（`MainPresenter.ReturnCardToHandOrBurnAsync` → `CardView.ResetRuntimeState`）で行う

カードの効果種別（`EventType`）の一覧と、キャラ・イベントカードへの設定方法は [event.md](event.md) を参照。

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コスト分だけ**手札から選んで墓地**へ送る。通常カードは1枚＝1コストだが、`CostBoost` カードは1枚で設定値（例: 2）コスト分として数える（コスト判定は枚数ではなく各カードの `CostPaymentValue` の合計で行う）。ただし `CostBoost` の倍化は**属性連動**：CostBoost カードの属性がプレイするカードの属性と一致するときのみ設定値分で、それ以外の属性のコストには通常どおり1として数える（白も一般属性として扱い、白 CostBoost は白のコストのみ倍化）。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで配置確定と同時 |
| イベントカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで確定と同時（即時解決） |

- コスト 0 のカードはコスト選択をスキップする（ドロップ後すぐ OK が有効）
- プレイするカード自身を除いた手札の支払い可能量（各カードの `CostPaymentValue` 合計）がコストに満たない場合はドロップを拒否し「コストが払えません」トーストを表示
- コスト選択 UI: カードをドロップするとコスト選択モードになり、クリックして選択（選択済カードは手札の扇から上に持ち上げて最前面に表示する演出。重なりから抜けるので選択状態が分かりやすく再クリックでの解除もしやすい）し OK で確定（1 回押すだけ）。選択カードの `CostPaymentValue` 合計がコストに達すると OK が有効
- カード詳細モーダル: カードをステージ済み（ドロップして OK/Back 確定待ち。コスト支払い中を含む）の間は、**手札カードをクリックしても詳細モーダルを開かない**（クリックはコスト選択として扱う／コスト 0 なら無反応）。Back で手札に戻せば再びクリックで詳細を閲覧できる。一方で**フィールド上のカード（プレイしようとしている当該カードを含む）はステージ中・コスト支払い中でもクリックで詳細を閲覧できる**
- CPU はプレイ前に支払い可能性を確認し（`CpuCanAffordCost`）、自身を除いた手札の `CostPaymentValue` 合計がコストに満たないカードは出さない。支払うカードは属性制約を満たすよう自動選択（同属性を優先して1枚確保、残りはコスト値の合計がコストに達するまで）
- オンライン対戦では払ったカードの ID をネットワーク送受信して双方の相手墓地表示を同期する

### 属性制約

全カードは `CardAttribute`（Red / Blue / Green / Yellow / Black / Purple / White）を持つ。白を含むすべての属性を同列に扱い、コスト支払い時に以下の制約がある：

| プレイするカードの属性 | コスト選択の制約 |
|---|---|
| 全属性（White を含む） | 選択したコストカードの中に「**同じ属性**」が **1 枚以上**必要 |

- 制約を満たさない場合は OK ボタンが無効になり、「「赤」のカードを1枚以上含めてください」のような警告を表示する
- White は万能コストではない。白カードは他属性のコストの属性要件を満たさない（コスト枚数の数合わせとしては選べる）
- 白カードをプレイするときは、コストに白カードを最低1枚含める必要がある

## 手札の上限

手札に持てる枚数の上限は **8枚（`HandView.MaxCards`）**。手札が満杯（8枚）の状態でカードが手札へ入るとき、そのカードは手札に加えず**墓地へ送る（バーン）**。バーンされたカードは墓地（公開情報）に表向きで積まれる。

上限判定とバーンは、手札が増える全経路で共通に適用する。

| 経路 | 満杯時の挙動 |
|---|---|
| ドローフェーズ | デッキから引いたうえで墓地へ送る（デッキ枚数は減る）。自分のドロー時のみ「手札が上限 → 墓地へ」トースト表示 |
| `EventType.Draw` 系効果 | 同上（引いて墓地へ） |
| `EventType.Bounce` / `BounceAll`（相手キャラを所有者の手札へ戻す） | 所有者の手札が満杯なら所有者の墓地へ送る。`BounceAll` は事前に空きスロット数を確定し、超過分をバーンへ振り分ける（同時実行の競合回避） |
| `EventType.Switch`（自キャラを手札へ戻す） | 戻し先の手札が満杯なら所有者の墓地へ送る |

- 共通のバーン演出は `MainPresenter.BurnCardToGraveyardAsync`（既存 CardView 用）／`BurnDrawnCardAsync`（ドロー由来の CardData 用）。手札へ戻すかバーンするかの分岐は `ReturnCardToHandOrBurnAsync` に集約する。
- オンライン対戦では相手デッキの内容が `OnlineInitialState.OpponentDeck` で双方に共有済みのため、両クライアントが同じカードを同じ墓地へ積み、表示が同期する（追加のネットワークメッセージ不要）。
- マリガンの初期手札（先攻3枚・後攻5枚）や Back ボタンでのコスト選択キャンセル（直前まで手札にあったカードを戻す）は上限を超えないため対象外。

## 勝敗条件

属性に依らない**共通の3条件**で勝敗が決まる（ゲーム開始時から両プレイヤーに適用。`WinRule` / 勝因種別は `WinReason`）。いずれか1つでも満たした瞬間に決着する。

### 1. デッキ切れ＝オーバーリミット（`WinReason.DeckOut`）

デッキが**0枚の状態でカードを引く／ミルしようとした瞬間**に、その本人が敗北＝**相手**が勝利する。**デッキを0枚にした引き／ミルそのものでは負けない**（0枚になった後、さらに引く／ミルする手番で初めて敗北する＝オーバーリミット）。

| 項目 | 内容 |
|---|---|
| 敗北の瞬間 | デッキが空（0枚）の状態で1枚引く／ミルしようとした直前 |
| ターン最初のドロー | **例外。常に安全**。デッキ枚数分だけ引いて止まり（デッキ1枚なら1枚引いてデッキ0、0枚なら0枚引く）、決して敗北しない |
| カードを引く効果（`ApplyDrawEffectAsync`） | 1枚引くごとに直前判定。2枚引く効果でデッキ1枚なら、1枚引いてデッキ0、2枚目を引こうとした瞬間に敗北 |
| ミル＝デッキ攻撃／デッキダメージ効果（`MillDeckAsync`） | 1枚ミルするごとに直前判定。デッキ攻撃は ATK が相手デッキ枚数を超えるとき敗北。空デッキへの `DamageEnemyDeck` / `DamageBothDecks` でも敗北を誘発できる |
| 判定 | カードを引く／ミルする**直前**に `CheckDeckOutWin`（`WinRule.IsDeckOut` ＝デッキが空か）で判定。空なら本人の敗北を成立させる |
| リミットブレイク告知 | デッキが**初めて0枚になった瞬間**（>0→0、＝オーバーリミット状態に入った瞬間）に「**リミットブレイク！**」を画面中央に告知する。ドロー・ミルでカードを抜いた直後に `UpdateLimitBreak` で down-crossing を検出（同じ空状態では1回だけ。`Recover` 等で1枚以上に戻ればフラグが戻り再検出）し、その**一連の処理を生き残った場合のみ**末尾で `PlayLimitBreakAnnouncementAsync` を再生する。**リミットブレイク直後に同じ処理内で敗北する場合（残り1枚に2ダメージ等）は告知しない**（敗北演出のみ） |

- 敗北側のデッキは0枚のまま。デッキが0枚（オーバーリミット状態）のプレイヤーは、**相手のデッキ攻撃（ミル）**・**デッキダメージ効果**・**本人のドロー効果**のいずれでも敗北する（空デッキへの ATK 1 以上のデッキ攻撃は最初のミルで即敗北）
- オンライン対戦では既存のドロー同期（`NGS_Draw`）とデッキ攻撃同期（`NGS_MainAction` の `targetsDeck`）を流用し、デッキ枚数・デッキ順が両クライアントで同期されるため追加のネットワークメッセージなしで同じ勝敗に到達する

### 2. 勝利点20（`WinReason.VictoryPoints`）

勝利点が **20（`WinRule.VictoryPointsToWin`）以上**になった側が勝利する。勝利点カウンター（メダル + 数字）は**ゲーム開始時から両者に常時表示**される（`VictoryPointsView`）。

| 項目 | 内容 |
|---|---|
| 表示位置 | 自分 → **自分フィールドの左下** ／ 相手 → **相手フィールドの右上**。開始時から 0 で表示 |
| 加点 | 全カード共通の `VictoryPointBonus` 付帯値（効果＝EventType とは独立した加算フィールド）で、効果解決時に発動した側の勝利点に加算する。加点カードは全属性で作れる（属性による制限なし）。「勝利点を得るだけ」は `EventType=None` ＋ 付帯値で表現し、「効果＋勝利点」も付帯値の併用で作れる。`EventType.GainVPPerGreenGrave` は固定値ではなく、発動した側の墓地にある緑属性カードの枚数だけ加算する（墓地に緑がなければ加点 0） |
| 勝敗 | 勝利点が 20 以上になった瞬間にその側が勝利（`AddVictoryPoints` 内で判定） |
| 演出 | 発動カードの上にメダルフロート（得点数 1〜5 に応じた Medal1〜Medal5Icon。0 や 6 以上は Medal1Icon） → 勝利点カウンターで「+N」とメダルの弾み → 数字カウントアップ。演出後に勝利判定 |

- 勝利条件は共通で、勝利点を**得る手段**（`VictoryPointBonus` 付帯値）は全属性のカードで設定できる。属性ゲートはなく、開始時から両者にカウンターが表示される
- オンライン対戦では既存のプレイ同期（`NGS_MainAction`）で効果が両クライアントに伝わり、勝利点も決定的に加算されるため追加のネットワークメッセージは不要

### 3. キャラ8体（`WinReason.FieldChars`）

自フィールドにキャラを **8体（`WinRule.FieldCharsToWin`）同時**に並べた側が勝利する。

| 項目 | 内容 |
|---|---|
| 勝敗 | 自フィールドのキャラ数が 8 体に達した瞬間にその側が勝利（破壊されて減ると再び 8 体並べ直す必要がある） |
| 判定タイミング | キャラ配置・召喚（通常配置・SummonChar・SummonFromDeckByKeyword・CopyFieldChar・Switch / Evolve での配置）の直後に `OnCardPlayed` → `CheckFieldCharsWin` で判定 |
| フィールド上限 | フィールドのキャラ上限は 9 体（`FieldView.MaxCharacters`）のため 8 体は到達可能 |

- オンライン対戦では配置・召喚が `NGS_MainAction` と決定的な召喚処理で両クライアントに同期されるため、キャラ数判定も対称に成立する（追加のネットワークメッセージ不要）

### 勝敗演出

勝敗決定時に画面中央にオーバーレイ演出を表示し、ホームに戻るボタンが出現する。

| 決着の種類 | 表示テキスト | 紋章 | テキスト色 |
|---|---|---|---|
| デッキ切れ（`WinReason.DeckOut`） | 「デッキ切れ勝利」「デッキ切れ敗北」 | GraveIcon | 青系（敗北側は暗め） |
| 勝利点（`WinReason.VictoryPoints`） | 「勝利点勝利」「勝利点敗北」 | Medal1Icon | 金系（敗北側は暗め） |
| キャラ8体（`WinReason.FieldChars`） | 「制圧勝利」「制圧敗北」 | AttackIcon | 橙系（敗北側は暗め） |
| 降参・タイムアウト・引き分け | 「YOU WIN」「YOU LOSE」「DRAW」 | なし | 勝=金 / 敗=赤 / 分=灰 |

- 降参勝ちは「対戦相手が降参しました」、自分の降参は「降参しました」をサブラベルに表示する
- 勝利時は花火演出、敗北時は次回ホームで雨演出（`ShouldRainOnNextHome`）
- オーバーレイの暗幕は `PickingMode.Ignore` でクリックを透過するため、勝敗演出後も**フィールドの表向きキャラカードをクリックしてカード詳細を閲覧できる**（再戦・ホームのボタンは通常どおりクリック可能）

#### 再戦

勝敗演出のオーバーレイには「再戦する」「ホームに戻る」の2ボタンが並ぶ（勝ち・負け・降参いずれの決着でも表示）。

- **オンライン**: 「再戦する」を押すと `NGS_Rematch` を送信し「対戦相手を待っています...」を表示。双方が押す（自分の希望と相手の希望が揃う）と両者が Main シーンを再ロードして新規対戦を開始する。相手が先に希望すると「対戦相手が再戦を希望しています」を表示。待機中に相手が退出すると「対戦相手が退出しました」を表示して再戦ボタンを消す
- **CPU 戦**: 「再戦する」を押すと相手待ちなしで即再戦
- 再戦はデッキ（`DeckModel`）を維持したまま再配牌し、先攻後攻はホストが再抽選する。NGO セッションは維持されるため再ハンドシェイク（`PrepareDecksAsync`）で開始する。詳細は [networking.md](networking.md#再戦ngs_rematch) を参照
