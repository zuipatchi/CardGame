# ゲームルール

## デッキ構築ルール

### コスト制限

| 項目 | 制限 |
|---|---|
| 推奨コスト | ぴったり 30（`DeckModel.TargetCost`）。超過してもカードを追加できる |
| コストオーバー | 合計が 30 を超えると「コストオーバー」が赤字表示され、コスト数字も赤くなる。Home シーンで対戦（CPU）・オンラインボタンを押すとトーストを表示して遷移をブロックする |
| 同名カードの枚数制限 | なし（同じカードを何枚でも入れられる） |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（コスト合計は常に表示される）
3. コスト合計が 30 を超えると「コストオーバー」が赤字で表示される
4. 「保存」ボタン（デッキ枚数 > 0 のとき有効）でデッキを保存する
5. 「もどる」を押すと Home シーンに戻る

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckRepository`（PlayerPrefs キー `"SavedCpuDeck"`）から読み込む
  - 保存済みデッキがある場合：そのカードIDリストで `CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - 保存済みデッキがない場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）

## ゲーム開始時フロー（1回のみ）

```
マリガンチェック（プレイヤー）
  ↓ 初期手札にキャラカードがなければ「MULLIGAN」告知 → 手札を全てデッキへ返却 → リシャッフル → 再ドロー
  ↓ （1回限り。再ドロー後にもキャラがなくても続行）
マリガンチェック（CPU）
  ↓ 同上の処理を CPU 側でも実施
  ↓
キャラセットフェーズ
  ↓ 先攻プレイヤーからキャラカードを1枚ずつ裏向きでスロットに置く（なければパス可）
  ↓ 2人とも配置または Pass したら Resolve
  ↓ 両スロットのキャラカードを同時に表向きにする
  ↓
通常ターンループへ
```

## ターンフロー

```
ドローフェーズ
  ↓ ターンプレイヤーがデッキから1枚ドロー
  ↓
戦闘前1フェーズ
  ↓ 先攻から順にキャラまたは技カードを1枚だけ裏向きでフィールドにセット（パス可）
  ↓
戦闘前2フェーズ
  ↓ ターンプレイヤーと相手が交互にイベントカードを手札からReadyにする
  ↓ 2連続パスで終了
  ↓
解決フェーズ
  ↓ Readyになった順の逆順でイベントカードを処理する（チェーン処理）
  ↓
戦闘フェーズ
  ↓ 全フィールドカードを表向き
  ↓ キャラカード → スロットへ移動（DEF適用）
  ↓ ATK/DEFカウントアップ演出（フィールドにATKアイコン・デッキ上にDEFアイコン）
  ↓ ATK数字 + 技カード → キャラスロットへ同時飛翔 → スロットにATKアイコン表示 → 技カードは墓地へ
  ↓ スロットのキャラが突撃（ATK > 0 かつ 今ターン新キャラ未配置の場合のみ・ATKアイコン同伴）
  ↓
相手のターンへ
```

## キャラセットフェーズの詳細

1. ゲーム開始時に**1回だけ**実行される
2. 先攻プレイヤーからキャラカードを1枚**裏向き**で自スロットに置く（スキップ可）
3. 後攻プレイヤーも同様に置く（スキップ可）
4. 両者の配置が完了したら Resolve: 両スロットのキャラカードを**同時に表向き**にする
5. Resolve 後、通常ターンループが開始する

---

## 戦闘前1フェーズの詳細

1. 先攻プレイヤーから順にキャラカードまたは技カードを1枚**裏向き**で自フィールドにセットする（スキップ可）
2. 後攻プレイヤーも同様にセットする（スキップ可）
3. セットしたカードは戦闘フェーズまで裏向きのまま

---

## 戦闘前2フェーズの詳細

1. ターンプレイヤーがイベントカードを手札から1枚 Ready 状態にする（スキップ可）
2. 相手プレイヤーがイベントカードを手札から1枚 Ready 状態にできる（スキップ可）
3. 相手が Ready にした場合、ターンプレイヤーが再び Ready の権利を得る
4. **2連続パス**したら戦闘前2フェーズ終了

---

## 解決フェーズの詳細

- Ready になった順の**逆順**でイベントカードを処理する（遊戯王のチェーンに相当）
- カードの対象は Ready になった時点で決定済み
- 処理後、全イベントカードは墓地へ送られる

### イベント効果一覧

| EffectType | 効果 | EffectValue の意味 |
|---|---|---|
| AtkBoost | 次の戦闘フェーズの ATK に加算 | 加算値 |
| DefBoost | 次の戦闘フェーズの有効 DEF に加算 | 加算値 |
| Draw | 発動プレイヤーが指定枚数デッキ上から手札に加える。1枚引くごとにデッキが 0 になるか確認し、0 になった時点でゲームオーバー判定・残りのドローを中断 | ドロー枚数 |

---

## 戦闘フェーズの詳細

1. 全フィールドカードを**同時に表向き**にする
2. キャラカードがフィールドにある場合、**自スロットへ移動**する（既存キャラは墓地へ）
3. ATK/DEFカウントアップ演出を表示する（フィールドにATKアイコン＋ATK値、デッキ上にDEFアイコン＋DEF値）
4. ATK数字ラベルと技カードが**同時にキャラスロットへ飛翔**する。スロット到着後にATKアイコン＋ATK値がキャラカード上に表示される
5. 技カードを墓地へ送る
6. ATK > 0 かつ **今ターン新キャラを配置していない** 場合、スロットのキャラが相手デッキへ突撃する（ATKアイコン同伴）
7. 突撃後にATKアイコンとDEFアイコンを非表示にする
8. ダメージ計算を行い、相手デッキからカードを墓地へ送る

### ATK の計算

- ATK = **(技ダメージ合計 + AtkBoost) × タイプ倍率 × 弱点倍率**
  - タイプ倍率 = **2**：技カードの属性 == 自スロットのキャラ属性（`None` 以外）
  - タイプ倍率 = **1**：それ以外（属性不一致・どちらかが `None`）
  - 弱点倍率 = **3**：技カードの属性 == 相手キャラの弱点属性（`AttributeDatabaseSO` で設定。`None` 以外）
  - 弱点倍率 = **1**：それ以外
  - AtkBoost はタイプ倍率・弱点倍率の**適用前**に加算する
  - 最大倍率：タイプ一致 ×2 × 弱点 ×3 = **×6**
- 以下の場合は ATK = 0（突撃モーションなし）:
  - キャラスロットにキャラカードがいない
  - 今ターンの戦闘前1でキャラカードをフィールドに出した（新キャラ配置ターン）

### ダメージ計算

- ATK 合計が相手スロットのキャラカードの**有効 DEF**（DEF + DefBoost）を**上回った場合**、その差分（ATK - 有効DEF）の枚数だけ相手デッキの上からカードを墓地へ送る
- ATK 合計が有効 DEF 以下の場合はダメージなし
- 相手スロットにキャラカードがいない場合、DEF = 0 として計算する
- 両プレイヤーのダメージを同時に計算する（先攻後攻なし）

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コストの枚数分だけ**自分のデッキ上から墓地**へカードを送る。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（キャラセット） | Resolve 後、カードが表向きになった直後 |
| キャラ・技カード（戦闘前1） | 戦闘フェーズで全フィールドカードが表向きになった直後 |
| イベントカード（戦闘前2） | OK ボタンで Ready を確定した直後 |

- コスト 0 のカードは墓地送りアニメーションをスキップする
- デッキ残枚数がコスト未満の場合は残り全枚数を払う
- コスト払い時にデッキが既に 0 枚（かつコスト > 0）の場合、その時点で敗北判定

---

## カード状態

| 状態 | 説明 |
|---|---|
| Normal | フィールド上で待機中 |
| Ready | 解決フェーズで処理される待機状態。対象はすでに決定済み（視覚的な区別なし） |
| Resolve | 解決フェーズで現在処理中のカード。黄色枠で表示 |

## 勝敗条件

以下のいずれかが発生した時点で敗北。

| 発生タイミング | 条件 |
|---|---|
| ドローフェーズ開始時 | 自分のデッキが 0 枚 |
| コスト支払い時（キャラセット・戦闘前1・戦闘前2・戦闘フェーズ） | 自分のデッキが 0 枚 かつ コスト > 0 |
| Draw エフェクトで次の 1 枚を引こうとした時 | 自分のデッキが 0 枚 |
| 戦闘フェーズ・ダメージ計算後 | ダメージを受けて自分のデッキが 0 枚になった |

- 上記以外のタイミングでデッキが 0 枚になること自体は即座に敗北にならない
- 勝敗決定時に画面中央に「YOU WIN」「YOU LOSE」「DRAW」の演出を表示し、タイトルに戻るボタンが出現する
