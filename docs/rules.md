# ゲームルール

## デッキ構築ルール

### 枚数制限

| 項目 | 制限 |
|---|---|
| デッキ枚数 | ちょうど 30 枚（`DeckModel.MaxCards`）でのみ対戦可能 |
| コスト上限 | なし（コスト合計に制限はない） |
| 枚数オーバー | 30 枚を超えると「30枚を超えています」が赤字表示され、枚数カウンターも赤くなる |
| Home での対戦ブロック | 枚数が 30 枚でないときにトーストを表示して遷移をブロック |
| 同名カードの枚数制限 | なし（同じカードを何枚でも入れられる） |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（フィルタボタン横に「枚数 N/30」が常時表示。コスト表示はなし）
3. 30 枚を超えると「30枚を超えています」が赤字で表示される
4. 「もどる」を押すと Home シーンに戻る（変更は自動保存済み）

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckSO`（ScriptableObject・Addressables キー `"Card/CpuDeck"`）から読み込む
  - カードIDリストが 1 件以上の場合：`CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - カードIDリストが空の場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）。編集内容は `CpuDeckSO`（`Assets/Data/CpuDeck.asset`）に保存され、ビルドに含まれる

## ゲーム開始時フロー（1回のみ）

```
マリガン選択（プレイヤー）
  ↓ 手札内容に関わらず YES/NO モーダルを表示
  ↓ YES → 「マリガン」告知 → 手札を全てデッキへ返却 → リシャッフル → 再ドロー
  ↓ NO → 手札をそのまま継続
  ↓ （1回限り）
マリガンチェック（CPU）
  ↓ 初期手札にキャラカードがなければ手札を全てデッキへ返却 → リシャッフル → 再ドロー（告知なし）
  ↓ （オンライン時は相手プレイヤーの決定を受信してアニメーション表示。下記参照）
  ↓
先攻後攻コイントス
  ↓ ランダム（オンラインはホスト決定）で先攻・後攻を決定
  ↓ コインフリップ演出 → 「先攻！」または「後攻！」を表示
  ↓ 後攻プレイヤーが手札を1枚追加ドロー
  ↓
通常ターンループへ（先攻プレイヤーのドローフェーズから開始）
```

**オンライン時のマリガン**: 両プレイヤーが独立して YES/NO モーダルで選択する。自分の選択後に `NGS_Mulligan` メッセージで決定を相手に送信し、相手の決定を受信するまで「対戦相手を待っています」を表示する。相手が YES を選択していた場合はアニメーション演出を表示する。マリガン要否の判定はホストが行わず、各プレイヤーが自律的に決定する。

## ターンフロー

ターンは先攻プレイヤーから交互に繰り返す。

```
【アクティブプレイヤーのターン開始】
ドローフェーズ
  ↓ アクティブプレイヤーのみデッキ上から1枚ドロー（デッキが 0 枚の場合はスキップ）
  ↓ オンライン時: アクティブプレイヤーが NGS_Draw を送信、相手がアニメーション再生
  ↓
メインフェーズ
  ↓ アクティブプレイヤーが以下のいずれか1アクションを選択する
  ↓   キャラを出す: 手札のキャラカードをフィールドに表向きで配置 → 手札からコスト枚数選択して墓地へ
  ↓   イベントを使う: 手札のイベントカードをフィールドへ → 即時解決（手札コスト選択 → 効果 → 墓地）
  ↓   攻撃する: フィールドのキャラ1体から矢印をドラッグして相手キャラへ攻撃
  ↓   パスする: 何もしない
  ↓
ターン終了 → 相手のターンへ
```

---

## メインフェーズの詳細

アクティブプレイヤーが以下の**1アクション**を選択して実行する。

### アクション選択肢

| アクション | 操作方法 | 処理 |
|---|---|---|
| キャラを出す | 手札のキャラカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドに表向きで配置 → 手札からコスト枚数を選択して墓地へ送る（フィールドのキャラは9体まで。満杯時はドロップ拒否） |
| イベントを使う | 手札のイベントカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドへ配置 → 即時解決（手札コスト選択 → 効果 → 墓地）|
| 攻撃する | フィールドのキャラから矢印をドラッグして相手キャラへ | 攻撃実行（下記「攻撃の詳細」参照）|
| パス | パスボタン | 何もしない（「パス」告知を表示） |

- コスト 0 のカードはドロップ後すぐ OK ボタンが有効になる
- Back ボタンでコスト選択をキャンセルしてカードを手札に戻せる
- AutoOk が有効な場合、コスト 0 はドロップ直後に自動で OK、コストありは必要枚数を選択した時点で自動で OK

### CPU のアクション選択順

1. プレイヤーのハートが攻撃可能で、自フィールドに ATK 1 以上のキャラがいればハートを攻撃（`CpuAgent.ChooseHeartAttacker`）
2. 自フィールドにキャラがいて、かつ相手フィールドにもキャラがいれば攻撃（`CpuAgent.ChooseBattleAttack`）
3. 手札にキャラカードがあれば配置
4. 手札にイベントカードがあれば使用
5. 上記いずれもなければパス

### 攻撃の詳細

- **一方向攻撃のみ**（カウンター攻撃なし）
- 攻撃対象: 相手フィールドの表向きキャラ、またはハートアイコン（矢印で選択。「勝敗条件」参照）
- 相手にキャラがいない場合はキャラへの攻撃は不可（ハートが出現していればハートへの攻撃は可能）

### ダメージ計算

- ダメージ = `ATK`（防御力なし）
- ダメージ > 0: 攻撃対象キャラの現在 HP からダメージ値を減算する（累積ダメージ）
- ダメージ = 0: シールドアイコンをフローティング表示して攻撃を終了

### キャラ破壊

- 攻撃を受けたキャラの現在 HP からダメージ値を減算する（累積ダメージ）
- HP が **0 以下**になったキャラカードを墓地へ送る
- HP の残量は `CardView.CurrentHp` で管理し、カード上の HP ラベルにリアルタイムで反映される
- キャラをフィールドに配置するたびに HP は最大値（`CardData.Hp`）にリセットされる（Switch で手札に戻して再出しした場合も同様）

### イベント効果一覧

カードの効果種別（`EventType`）の一覧と、キャラ・イベントカードへの設定方法は [event.md](event.md) を参照。

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コスト分だけ**手札から選んで墓地**へ送る。通常カードは1枚＝1コストだが、`CostBoost` カードは1枚で設定値（例: 2）コスト分として数える（コスト判定は枚数ではなく各カードの `CostPaymentValue` の合計で行う）。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで配置確定と同時 |
| イベントカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで確定と同時（即時解決） |

- コスト 0 のカードはコスト選択をスキップする（ドロップ後すぐ OK が有効）
- プレイするカード自身を除いた手札の支払い可能量（各カードの `CostPaymentValue` 合計）がコストに満たない場合はドロップを拒否し「コストが払えません」トーストを表示
- コスト選択 UI: カードをドロップするとコスト選択モードになり、クリックして選択（選択済カードは透明度低下）し OK で確定（1 回押すだけ）。選択カードの `CostPaymentValue` 合計がコストに達すると OK が有効
- CPU は属性制約を満たすよう自動選択（同属性 or White を優先して1枚確保、残りはコスト値の合計がコストに達するまで）
- オンライン対戦では払ったカードの ID をネットワーク送受信して双方の相手墓地表示を同期する

### 属性制約

全カードは `CardAttribute`（Red / Blue / Green / Yellow / Black / Purple / White）を持つ。コスト支払い時に以下の制約がある：

| プレイするカードの属性 | コスト選択の制約 |
|---|---|
| White 以外 | 選択したコストカードの中に「同じ属性 OR White」が **1 枚以上**必要 |
| White | 任意の属性のカードでコストを払える（制約なし） |

- 制約を満たさない場合は OK ボタンが無効になり、「「赤」のカードを1枚以上含めてください」のような警告を表示する
- White カードはどの属性のコストとしても使える（万能コスト）

## 勝敗条件

### ハート勝利条件

赤属性カードを1度でも**プレイ**（キャラとして出す・イベントとして使う。Switch / Evolve 効果によるキャラ配置も含む）すると、プレイした側の攻撃対象として相手側にハートアイコン3個が出現する（`HeartRule` / `LifeHeartsView`）。

| 項目 | 内容 |
|---|---|
| 出現位置 | 自分が赤をプレイ → **相手フィールドの右上**に3個 ／ 相手が赤をプレイ → **自分フィールドの左下**に3個 |
| 出現条件 | 赤属性カードのプレイ（コスト支払いで赤を墓地に送ってもカウントしない） |
| 永続性 | 一度出現したら消えない（赤カードがフィールド・墓地から無くなっても表示は維持） |
| 攻撃方法 | メインフェーズの攻撃アクションとして、フィールドのキャラから矢印をドラッグしてハートへ。相手キャラの有無に関係なくいつでも攻撃可能 |
| ダメージ | ATK 1 以上でハート1個破壊（ATK 値は破壊数に影響しない）。ATK 0 はシールドアイコン表示で不発 |
| 被弾演出 | 突進 → パーティクル再生 → ハート削除 |
| 勝敗 | ハート3個すべて破壊した側が勝利（破壊された側は敗北） |

- CPU はハートが攻撃可能なら最優先でハートを攻撃する（ATK 1 以上の最高 ATK キャラを使用）
- オンライン対戦ではハート攻撃を `NGS_MainAction`（`targetsHeart`）で同期する

### 青属性の勝利条件（デッキ0勝利）

青属性カードを1度でも**プレイ**（キャラとして出す・イベントとして使う。Switch / Evolve 効果によるキャラ配置も含む）すると、プレイした側の「自分のデッキが0枚になったら勝利」が武装される（`BlueRule`）。

| 項目 | 内容 |
|---|---|
| 武装条件 | 青属性カードのプレイ（コスト支払いで青を墓地に送ってもカウントしない） |
| 表示 | 武装した側の自デッキ枚数表示に `WaterIcon` を重ねて表示（`DeckView.ShowBlueWinIcon`） |
| 永続性 | 一度武装したら解除されない（青カードがフィールド・墓地から無くなっても維持） |
| 勝敗 | 武装済みで**自分のデッキが0枚**になった瞬間にそのプレイヤーが勝利（相手は敗北） |
| 判定タイミング | デッキが減る各ドロー処理（ドローフェーズ・後攻補正ドロー・Drawイベント効果）の直後に `CheckBlueWin` で判定 |
| 未武装時 | 青を1枚もプレイしていない側のデッキが0になっても何も起きない（ドローをスキップするのみ） |

- デッキが減るのは「ドロー」のみのため、上記のドロー処理直後の判定で漏れなく成立する
- オンライン対戦では既存のドロー同期（`NGS_Draw`）とプレイ同期（`NGS_MainAction`）を流用し、追加のネットワークメッセージなしで両クライアントが同じ勝敗に到達する

### 緑属性の勝利条件（勝利点）

緑属性カードを1度でも**プレイ**（キャラとして出す・イベントとして使う。Switch / Evolve 効果によるキャラ配置も含む）すると、プレイした側に勝利点表示（MedalIcon + 数字）が出現する（`GreenRule` / `VictoryPointsView`）。

| 項目 | 内容 |
|---|---|
| 出現条件 | 緑属性カードのプレイ（コスト支払いで緑を墓地に送ってもカウントしない） |
| 出現位置 | 自分が緑をプレイ → **自分フィールドの左下** ／ 相手が緑をプレイ → **相手フィールドの右上**（ハートの内側にオフセット） |
| 表示 | プレイした側のみに MedalIcon + 数字を表示（緑を使っていない側には出ない） |
| 永続性 | 一度出現したら消えない |
| 加点 | `EventType.GainVictoryPoints`（EventValue = 加算点）の効果で、発動した側の勝利点に加算する。加点カードは緑属性で作る |
| 勝敗 | 勝利点が **20（`GreenRule.WinPoints`）以上**になった側が勝利（相手は敗北） |
| 演出 | 発動カードの上に MedalIcon フロート → 勝利点カウンターで「+N」とメダルの弾み → 数字カウントアップ。演出後に勝利判定 |

- オンライン対戦では既存のプレイ同期（`NGS_MainAction`）で効果が両クライアントに伝わり、勝利点も決定的に加算されるため追加のネットワークメッセージは不要

### 勝敗演出

勝敗決定時に画面中央にオーバーレイ演出を表示し、ホームに戻るボタンが出現する。

| 決着の種類 | 表示テキスト | 紋章 | テキスト色 |
|---|---|---|---|
| 色による勝利条件（赤=ハート / 青=デッキ0 / 緑=勝利点20） | 「赤色勝利」「緑色敗北」など（勝因の色名 + 勝利/敗北） | 勝因属性のアイコン（赤=Fire / 青=Water / 緑=Medal…）をスケールイン表示 | 勝因の属性色（敗北側は暗め） |
| 降参・タイムアウト・引き分け | 「YOU WIN」「YOU LOSE」「DRAW」 | なし | 勝=金 / 敗=赤 / 分=灰 |

- 降参勝ちは「対戦相手が降参しました」、自分の降参は「降参しました」をサブラベルに表示する
- 勝利時は花火演出、敗北時は次回ホームで雨演出（`ShouldRainOnNextHome`）
