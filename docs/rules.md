# ゲームルール

## デッキ構築ルール

### コスト制限

| 項目 | 制限 |
|---|---|
| 推奨コスト | ぴったり 50（`DeckModel.TargetCost`）。超過してもカードを追加できる |
| コストオーバー | 合計が 50 を超えると「コストオーバー」が赤字表示され、コスト数字も赤くなる。Home シーンで対戦（CPU）・オンラインボタンを押すとトーストを表示して遷移をブロックする |
| 同名カードの枚数制限 | なし（同じカードを何枚でも入れられる） |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（コスト合計は常に表示される）
3. コスト合計が 50 を超えると「コストオーバー」が赤字で表示される
4. 「保存」ボタン（デッキ枚数 > 0 のとき有効）でデッキを保存する
5. 「もどる」を押すと Home シーンに戻る

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckSO`（ScriptableObject・Addressables キー `"Card/CpuDeck"`）から読み込む
  - カードIDリストが 1 件以上の場合：`CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - カードIDリストが空の場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）。編集内容は `CpuDeckSO`（`Assets/Data/CpuDeck.asset`）に保存され、ビルドに含まれる

## ゲーム開始時フロー（1回のみ）

```
マリガンチェック（プレイヤー）
  ↓ 初期手札にキャラカードがなければ「マリガン」告知 → 手札を全てデッキへ返却 → リシャッフル → 再ドロー
  ↓ （1回限り。再ドロー後にもキャラがなくても続行）
マリガンチェック（CPU / 相手プレイヤー）
  ↓ 同上の処理を CPU 側でも実施（オンライン時は相手のマリガンをアニメーション演出で表示）
  ↓
攻撃優先権コイントス
  ↓ ランダム（オンラインはホスト決定）でどちらが優先権を得るか決定
  ↓ コインフリップ演出 → 「優先権 獲得！」または「優先権 なし」を表示
  ↓ 優先権を得られなかったプレイヤーが手札を1枚ドロー
  ↓ 優先権保持者の墓地隣にコインアイコンを表示
  ↓
通常ターンループへ（1ターン目の Draw → キャラセット から開始）
```

**オンライン時のマリガン**: ホストがデッキ配布時に両プレイヤーの手札を確認してマリガン要否を判定し、`NGS_InitialState` に `localNeedsMulligan` / `opponentNeedsMulligan` フラグとして含める。各クライアントは受信したフラグに基づき、自分のマリガンを実行し、相手がマリガンする場合はアニメーション演出を表示する。

## ターンフロー

```
ドローフェーズ
  ↓ 両プレイヤーが同時にデッキから1枚ドロー
  ↓
キャラセットフェーズ
  ↓ 両スロットが埋まっている場合はスキップ
  ↓ スロットが空のプレイヤーのみキャラカードを1枚裏向きで配置できる（パス可）
  ↓ スロットが埋まっているプレイヤーは強制パス
  ↓ Resolve → 新たに置かれたカードのみ表向き＋コスト払い
  ↓
戦闘前1フェーズ
  ↓ 両プレイヤーが同時にスキルカードを1枚だけ裏向きでフィールドにセット（パス可）
  ↓ キャラカードは出せない
  ↓
戦闘前2フェーズ（イベントフェーズ）
  ↓ スロットのキャラカードの Speed を比較して先攻後攻を決定（"YOUR TURN"/"ENEMY TURN" 表示）
  ↓ Speed が高い方が先攻。同値の場合: 初回ターンはランダム・以降は交互
  ↓ 先攻プレイヤーから交互にイベントカードを手札からReadyにする
  ↓ 2連続パスで終了
  ↓ Readyになった順の逆順でイベントカードを処理する（解決・チェーン処理）
  ↓
戦闘フェーズ
  ↓ 全フィールドカードを表向き
  ↓ ATK/DEFカウントアップ演出（フィールドにATKアイコン・キャラスロットにDEFアイコン）
  ↓ 先攻スキルカード → 相手キャラスロットへ突撃 → 先攻ダメージ適用 → ダメージ ≥ HP なら相手キャラ破壊 → 後攻スキルカード → 先攻キャラスロットへ突撃 → 後攻ダメージ適用 → ダメージ ≥ HP なら後攻キャラ破壊 → スキルカードは墓地へ
  ↓
次のターンへ
```

## キャラセットフェーズの詳細

毎ターン、ドローフェーズの直後に実行される。

1. **両スロット埋まりの場合はスキップ**：両プレイヤーのキャラスロットにカードがある場合、このフェーズ全体を省略する
2. **片方または両方のスロットが空の場合に実行**
   - スロットが空のプレイヤー：キャラカードを1枚裏向きで自スロットに置く（パス可）
   - スロットが埋まっているプレイヤー：自動的にパス（入力不要）
   - CPU戦: `UniTask.WhenAll` で自分と CPU の選択を並列実行
   - オンライン: 相手カード受信タスクを並列起動し、自分の入力・送信後に相手受信完了を await
3. 両者の配置・パスが完了したら Resolve アニメーション
4. Resolve 後、**新たに置かれたカードのみ**表向きにしてコストを払う（フェーズ開始前から埋まっていたスロットはスキップ）

---

## 戦闘前1フェーズの詳細

1. **両プレイヤーが同時**にスキルカードを1枚**裏向き**で自フィールドにセットする（スキップ可）。キャラカードは出せない
   - CPU戦: `UniTask.WhenAll` で自分と CPU の選択を並列実行
   - オンライン: キャラセットフェーズと同様の対称プロトコルで並列実行
2. セットしたカードは戦闘フェーズまで裏向きのまま

---

## 戦闘前2フェーズ（イベントフェーズ）の詳細

1. **先攻後攻決定**: スロットのキャラカードの Speed を比較する（キャラなし = Speed 0）
   - Speed が高い方が先攻 → "YOUR TURN" / "ENEMY TURN" を表示
   - Speed が同値の場合: **攻撃優先権**を持つプレイヤーが先攻。行使後は優先権が相手に移る
   - 優先権はゲーム開始時のコイントスで決定（オンラインはホストが決定した `OnlineInitialState.IsLocalFirst` を使用）
2. 先攻プレイヤーがイベントカードを手札から1枚 Ready 状態にする（スキップ可）
3. 相手プレイヤーがイベントカードを手札から1枚 Ready 状態にできる（スキップ可）
4. 相手が Ready にした場合、先攻プレイヤーが再び Ready の権利を得る
5. **2連続パス**したら戦闘前2フェーズ終了
6. 2連続パス後、**解決処理**を自動実行：Ready になった順の**逆順**でイベントカードを処理する（遊戯王のチェーンに相当）。カードの対象は Ready になった時点で決定済み。処理後、全イベントカードは墓地へ送られる

### イベント効果一覧

| EventType | 効果 | EventValue の意味 |
|---|---|---|
| AtkBoost | 次の戦闘フェーズの ATK に加算 | 加算値 |
| DefBoost | 次の戦闘フェーズの有効 DEF に加算 | 加算値 |
| Draw | 発動プレイヤーが指定枚数デッキ上から手札に加える。1枚引くごとにデッキが 0 になるか確認し、0 になった時点でゲームオーバー判定・残りのドローを中断 | ドロー枚数 |
| Negate | 解決フェーズで自分の直後に処理されるカードの効果を無効化する。無効化されたカード自体は墓地へ送られる。Negate が Negate の直後に処理されると、その Negate を無効化する（連鎖打ち消し） | 使用しない（0 固定） |
| BanishChar | 相手のキャラスロットにいるキャラカードを相手の墓地へ送る。スロットが空の場合は空振り | 使用しない（0 固定） |
| Recover | 発動プレイヤー自身の墓地の**上から**指定枚数を取り出し、自デッキに加えてシャッフルする。墓地枚数が指定枚数未満の場合は存在する全枚数を回収する | 回収枚数 |

## 戦闘フェーズの詳細

1. 全フィールドカードを**同時に表向き**にする
2. 素早さ同値で攻撃優先権が行使された場合：「優先権 行使！」アナウンスと同時にコインが画面中央へ飛翔し、アナウンス消去後に相手の墓地隣へ移動して優先権が移る
3. 先攻プレイヤーの「PAY X COSTS」アナウンス → コスト払い → 相手の「PAY X COSTS」アナウンス → コスト払い（コスト 0 の場合も PAY 0 COSTS を表示）
3. ATK/DEFカウントアップ演出を表示する（フィールドにATKアイコン＋ATK値、キャラスロットにDEFアイコン＋DEF値）
4. 先攻プレイヤーの**ATKカウンターアイコン**がフィールドから**相手キャラスロットへ突撃**する（予備動作 → 直線飛翔 → ノックバック）。ATK が 0 の場合は突撃せず「NO ATTACK」ラベルを表示する。ダメージ > 0 の場合は**ダメージアイコン＋ダメージ数字**が相手キャラスロット中央でスケールイン出現 → ホールド → 相手デッキへ飛翔 → 先攻のダメージを相手デッキに適用 → 相手デッキが 0 枚になったら先攻勝利で戦闘フェーズ終了 → **ダメージ ≥ キャラの HP なら相手キャラを墓地へ送る**（キャラ破壊）
5. 先攻が終わった後、後攻プレイヤーの ATKカウンターアイコンが同様に突撃・ダメージ飛翔（ATK が 0 の場合は「NO ATTACK」ラベルを表示）。後攻のダメージを先攻デッキに適用 → 先攻デッキが 0 枚になったら後攻勝利で戦闘フェーズ終了 → **ダメージ ≥ キャラの HP なら後攻キャラを墓地へ送る**（キャラ破壊）
6. スキルカードを墓地へ送る

### ATK の計算

- ATK = **キャラATK + スキルダメージ合計 + AtkBoost**
- 以下の場合は ATK = 0（突撃モーションなし・「NO ATTACK」グレーラベルを表示）:
  - キャラスロットにキャラカードがいない
  - 先攻の攻撃でキャラが破壊され、後攻の ATK が 0 に設定された場合も表示

### ダメージ計算

- ダメージ = `MAX(0, ATK - 有効DEF)`。有効DEF = DEF + DefBoost
- ダメージ枚数分だけ相手デッキの上からカードを墓地へ送る
- ATK が有効DEF以下の場合はダメージなし（0枚）
- 相手スロットにキャラカードがいない場合、有効DEF = 0（DefBoost も無効）として計算する

### キャラ破壊

- デッキダメージ適用後、**ダメージ ≥ キャラの HP** の場合、そのキャラカードを墓地へ送る
- デッキダメージとキャラ破壊は独立して発生する（どちらも同じターンに起こりうる）
- キャラが破壊されても次ターン以降にキャラセットで補充可能

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コストの枚数分だけ**自分のデッキ上から墓地**へカードを送る。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（キャラセット） | Resolve 後、カードが表向きになった直後 |
| キャラ・スキルカード（戦闘前1） | 戦闘フェーズで全フィールドカードが表向きになった直後 |
| イベントカード（戦闘前2） | OK ボタンで Ready を確定した直後 |

- コスト 0 のカードは墓地送りアニメーションをスキップする
- コスト > デッキ残枚数の場合は残り全枚数を払い切り、演出完了後に敗北判定
- コスト == デッキ残枚数の場合はデッキが 0 枚になるが敗北しない

---

## カード状態

| 状態 | 説明 |
|---|---|
| Normal | フィールド上で待機中 |
| Ready | 解決フェーズで処理される待機状態。対象はすでに決定済み（視覚的な区別なし） |
| Resolve | 解決フェーズで現在処理中のカード。黄色枠で表示 |

## 勝敗条件

以下のいずれかが発生した時点で敗北。

| 発生タイミング | 条件 |
|---|---|
| ドローフェーズ開始時 | 自分のデッキが 0 枚 |
| コスト支払い時（キャラセット・戦闘前1・戦闘前2・戦闘フェーズ） | 払うコストがデッキ残枚数より多い（コスト > デッキ残枚数）。演出で残り全枚数を払い切った後に敗北。コスト == デッキ残枚数の場合は敗北しない |
| Draw エフェクトで次の 1 枚を引こうとした時 | 自分のデッキが 0 枚 |
| 戦闘フェーズ・ダメージ計算後 | ダメージを受けて自分のデッキが 0 枚になった |

- 上記以外のタイミングでデッキが 0 枚になること自体は即座に敗北にならない
- 勝敗決定時に画面中央に「YOU WIN」「YOU LOSE」「DRAW」の演出を表示し、ホームに戻るボタンが出現する
