# ゲームルール

## デッキ構築ルール

### 枚数・コスト制限

| 項目 | 制限 |
|---|---|
| デッキ枚数 | ちょうど 30 枚（`DeckModel.MaxCards`）でのみ対戦可能 |
| コスト上限 | コスト合計が 80 以下（`DeckModel.MaxCost`）でのみ対戦可能 |
| 枚数オーバー | 30 枚を超えると「30枚を超えています」が赤字表示され、枚数カウンターも赤くなる |
| コストオーバー | コスト合計が 80 を超えると「コストが80を超えています」が赤字表示され、コストカウンターも赤くなる |
| Home での対戦ブロック | 枚数が 30 枚でない・コストが 80 超のいずれかでもトーストを表示して遷移をブロック |
| 同名カードの枚数制限 | なし（同じカードを何枚でも入れられる） |

### 構築手順

1. Home シーンから「デッキ構築」を押すとデッキ構築シーンへ遷移する
2. 左パネルのカード一覧から選択してデッキに追加する（フィルタボタン横に「枚数 N/30」「コスト N/80」が常時表示）
3. 30 枚を超えると「30枚を超えています」、コスト合計が 80 を超えると「コストが80を超えています」が赤字で表示される
4. 「もどる」を押すと Home シーンに戻る（変更は自動保存済み）

### その他

- デッキは DeckModel に保持され、DeckBuilder → Main 遷移をまたいで引き継がれる
- CPU デッキは `CpuDeckSO`（ScriptableObject・Addressables キー `"Card/CpuDeck"`）から読み込む
  - カードIDリストが 1 件以上の場合：`CardDatabase.BuildDeck` を呼び出したカード群をシャッフルして使用する
  - カードIDリストが空の場合：カードDB全カードをシャッフルした全枚数をフォールバックとして使用する
- CPU デッキの編集は Unity Editor 専用の `CpuDeckBuilder` シーンで行う（タイトル画面の「CPU デッキ編集」ボタン、`#if UNITY_EDITOR` でのみ表示）。編集内容は `CpuDeckSO`（`Assets/Data/CpuDeck.asset`）に保存され、ビルドに含まれる

## ゲーム開始時フロー（1回のみ）

```
マリガン選択（プレイヤー）
  ↓ 手札内容に関わらず YES/NO モーダルを表示
  ↓ YES → 「マリガン」告知 → 手札を全てデッキへ返却 → リシャッフル → 再ドロー
  ↓ NO → 手札をそのまま継続
  ↓ （1回限り）
マリガンチェック（CPU）
  ↓ 初期手札にキャラカードがなければ手札を全てデッキへ返却 → リシャッフル → 再ドロー（告知なし）
  ↓ （オンライン時は相手プレイヤーの決定を受信してアニメーション表示。下記参照）
  ↓
先攻後攻コイントス
  ↓ ランダム（オンラインはホスト決定）で先攻・後攻を決定
  ↓ コインフリップ演出 → 「先攻！」または「後攻！」を表示
  ↓ 後攻プレイヤーが手札を1枚追加ドロー
  ↓
通常ターンループへ（先攻プレイヤーのドローフェーズから開始）
```

**オンライン時のマリガン**: 両プレイヤーが独立して YES/NO モーダルで選択する。自分の選択後に `NGS_Mulligan` メッセージで決定を相手に送信し、相手の決定を受信するまで「対戦相手を待っています」を表示する。相手が YES を選択していた場合はアニメーション演出を表示する。マリガン要否の判定はホストが行わず、各プレイヤーが自律的に決定する。

## ターンフロー

ターンは先攻プレイヤーから交互に繰り返す。

```
【アクティブプレイヤーのターン開始】
ドローフェーズ
  ↓ アクティブプレイヤーのみデッキ上から1枚ドロー
  ↓ デッキが 0 枚の場合は即座にそのプレイヤーの敗北
  ↓ オンライン時: アクティブプレイヤーが NGS_Draw を送信、相手がアニメーション再生
  ↓
メインフェーズ
  ↓ アクティブプレイヤーが以下のいずれか1アクションを選択する
  ↓   キャラを出す: 手札のキャラカードをフィールドに表向きで配置 → 手札からコスト枚数選択して墓地へ
  ↓   イベントを使う: 手札のイベントカードをフィールドへ → 即時解決（手札コスト選択 → 効果 → 墓地）
  ↓   攻撃する: フィールドのキャラ1体から矢印をドラッグして相手キャラ or 相手デッキへ攻撃
  ↓   パスする: 何もしない
  ↓
ターン終了 → 相手のターンへ
```

---

## メインフェーズの詳細

アクティブプレイヤーが以下の**1アクション**を選択して実行する。

### アクション選択肢

| アクション | 操作方法 | 処理 |
|---|---|---|
| キャラを出す | 手札のキャラカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドに表向きで配置 → 手札からコスト枚数を選択して墓地へ送る |
| イベントを使う | 手札のイベントカードをフィールドへドロップ → コスト選択 → OK ボタン | フィールドへ配置 → 即時解決（手札コスト選択 → 効果 → 墓地）|
| 攻撃する | フィールドのキャラから矢印をドラッグして相手キャラ or 相手デッキへ | 攻撃実行（下記「攻撃の詳細」参照）|
| パス | パスボタン | 何もしない（「パス」告知を表示） |

- コスト 0 のカードはドロップ後すぐ OK ボタンが有効になる
- Back ボタンでコスト選択をキャンセルしてカードを手札に戻せる
- AutoOk が有効な場合、コスト 0 はドロップ直後に自動で OK、コストありは必要枚数を選択した時点で自動で OK

### CPU のアクション選択順

1. フィールドにキャラがいれば攻撃（`CpuAgent.ChooseBattleAttack`）
2. 手札にキャラカードがあれば配置
3. 手札にイベントカードがあれば使用
4. 上記いずれもなければパス

### 攻撃の詳細

- **一方向攻撃のみ**（カウンター攻撃なし）
- 攻撃対象: 相手フィールドの表向きキャラ（矢印で選択）、または相手デッキ直撃（矢印を相手デッキにドロップ）
- 相手にキャラがいない場合はデッキ直撃のみ可能

### ダメージ計算

- ダメージ = `MAX(0, ATK - DEF)`
- 攻撃対象がデッキ直撃（target = null）の場合、DEF = 0 として計算する
- ダメージ > 0: ダメージ数字アイコン演出 → 相手デッキ上からダメージ枚数分を墓地へ送る
- ダメージ = 0: 「NO DAMAGE」水色ラベルを表示して攻撃を終了
- 要求枚数を出し切れなかった場合は攻撃プレイヤーの勝利（相手の敗北）

### キャラ破壊

- デッキダメージ適用後、**ダメージ ≥ キャラの HP** の場合、そのキャラカードを墓地へ送る
- デッキダメージとキャラ破壊は独立して発生する

### イベント効果一覧

| EventType | 効果 | EventValue の意味 |
|---|---|---|
| Draw | 発動プレイヤーが指定枚数デッキ上から手札に加える。1枚引くごとにデッキが 0 になるか確認し、0 になった時点でゲームオーバー判定・残りのドローを中断 | ドロー枚数 |
| BanishChar | 相手フィールドの先頭キャラ（`Characters[0]`）を墓地へ送る。相手フィールドにキャラがいない場合は空振り | 使用しない（0 固定） |
| Recover | 発動プレイヤー自身の墓地の**上から**指定枚数を取り出し、自デッキに加えてシャッフルする。墓地枚数が指定枚数未満の場合は存在する全枚数を回収する | 回収枚数 |
| Switch | 発動プレイヤー自身のフィールドのキャラを1体選んで（複数いる場合はクリックで選択）手札に戻し、手札からキャラカードを1枚コストを払って配置する。フィールドにキャラがいない場合は効果なし | 使用しない（0 固定） |
| CharDamage | 相手キャラに直接ダメージを与える。ダメージ = EventValue − 相手キャラ DEF（0 未満は 0）。damage ≥ 相手キャラ HP でキャラ破壊＋デッキダメージ、0 < damage < HP でデッキダメージのみ、damage = 0 で NO DAMAGE。相手キャラなしの場合は EventValue がそのままデッキダメージ | ベース ATK 値 |
| Evolve | 自分のフィールドのキャラを1体選んで（複数いる場合はクリックで選択）墓地に送り、手札から犠牲キャラより高コストのキャラカードを1枚コストなしで配置する。フィールドにキャラがいないまたは手札に適格カードがない場合は何もしない。配置時にエフェクト再生 | 使用しない（0 固定） |
| Poison | 同ターンの攻撃でダメージ > 0 が入った場合、攻撃終了後に相手キャラを破壊する。ダメージが 0 の場合（DEF で全防御）は発動しない。相手キャラが攻撃で既に破壊されていれば追加破壊はなし。`_playerPoisoned` / `_opponentPoisoned` フラグで管理し、攻撃後に `ResetTurnState` でリセット | 使用しない（0 固定） |
| DeckMill | お互いのデッキ上から EventValue 枚を墓地へ送る。**発動プレイヤー自身のデッキを先に**処理し、次に相手デッキを処理する（相手が発動した場合は相手デッキ先 → 自デッキ後）。要求枚数を出し切れなかった場合に敗北が確定する。デッキ枚数がちょうど EventValue の場合は全枚ミルしても敗北にならない。ミルしたカードに `TriggerOnGrave` がある場合は墓地トリガーが発動する | ミル枚数 |
| BattleEndMill | **永続効果**。カードを発動したプレイヤーの相手デッキ上から、毎ターン攻撃終了後に EventValue 枚を墓地へ送る。発動時と毎ターンのトリガー時に毒パーティクルで演出。両プレイヤーが BattleEndMill を持つ場合は攻撃プレイヤー側の効果から処理する。要求枚数を出し切れなかった場合に敗北確定。`_localBattleEndMillValue` / `_opponentBattleEndMillValue` フィールドで永続管理し、`ResetTurnState` ではリセットしない。**同一プレイヤーが複数枚発動した場合、後から発動したカードの EventValue で上書き（累積しない）** | ミル枚数（毎ターン） |

### グレイブトリガー（EventCardData.TriggerOnGrave）

`EventCardData` に `TriggerOnGrave: bool` フラグを追加した特殊な効果。

- `TriggerOnGrave = true` のイベントカードは、そのカード自身がデッキから墓地に送られた瞬間に効果が発動する
- 発動タイミング：攻撃ダメージ（`PlayDeckDamageAsync` 経由）で送られたとき。手札コスト払いでは発動しない（手札から送られるため）
- 発動演出：カードのオーナー側の墓地近くに「TRAP！」フローティングラベルを表示してから効果を適用
- 効果は `EventType` と `EventValue` で定義し、通常のイベント効果（CharDamage / Draw / Recover など）をそのまま使用できる
- 同じ `PlayDeckDamageAsync` の中で複数の TriggerOnGrave カードが送られた場合は、送られた順にトリガーを処理する
- カードのオーナー（`CardView.IsOpponent`）からトリガー発動者の `isLocal` を決定する

## コストシステム

全カードは `Cost` 値を持つ。カードをプレイするとき、コストの枚数分だけ**手札から選んで墓地**へ送る。

| カード種別 | コストを払うタイミング |
|---|---|
| キャラカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで配置確定と同時 |
| イベントカード（メインフェーズ） | ドロップ直後にコスト選択UI表示 → コスト選択後の OK ボタンで確定と同時（即時解決） |

- コスト 0 のカードはコスト選択をスキップする（ドロップ後すぐ OK が有効）
- 手札 < コストの場合は払えるだけ払う（敗北にはならない）
- コスト選択 UI: カードをドロップすると即座に手札が黄ハイライト表示され、クリックして選択（赤ボーダー = 選択済）し OK で確定（1 回押すだけ）
- 手札が足りない場合は「手札が足りません（N/M枚）」と表示し、払えるN枚で確定
- CPU は属性制約を満たすよう自動選択（同属性 or White を優先して1枚確保、残りは任意）
- オンライン対戦では払ったカードの ID をネットワーク送受信して双方の相手墓地表示を同期する

### 属性制約

全カードは `CardAttribute`（Red / Blue / Green / Yellow / Black / Purple / White）を持つ。コスト支払い時に以下の制約がある：

| プレイするカードの属性 | コスト選択の制約 |
|---|---|
| White 以外 | 選択したコストカードの中に「同じ属性 OR White」が **1 枚以上**必要 |
| White | 任意の属性のカードでコストを払える（制約なし） |

- 制約を満たさない場合は OK ボタンが無効になり、「「赤」のカードを1枚以上含めてください」のような警告を表示する
- White カードはどの属性のコストとしても使える（万能コスト）

## カード状態

| 状態 | 説明 |
|---|---|
| Normal | フィールド上で待機中 |
| Ready | 解決フェーズで処理される待機状態。対象はすでに決定済み（視覚的な区別なし） |
| Resolve | 解決フェーズで現在処理中のカード。黄色枠で表示 |

## 勝敗条件

以下のいずれかが発生した時点で敗北。

| 発生タイミング | 条件 |
|---|---|
| ドローフェーズ開始時 | アクティブプレイヤーのデッキが 0 枚 |
| Draw イベント効果で次の 1 枚を引こうとした時 | 自分のデッキが 0 枚 |
| メインフェーズ・攻撃ダメージ計算後 | ダメージ量が相手デッキ残枚数を超えた（要求枚数を出し切れなかった場合）。ちょうどダメージ量と残枚数が同じ場合は敗北しない |
| CharDamage イベント効果後 | ダメージ量が自分のデッキ残枚数を超えた（要求枚数を出し切れなかった場合） |
| DeckMill イベント効果後 | お互いのデッキで要求枚数を出し切れなかった側が敗北（発動プレイヤーのデッキを先処理） |
| BattleEndMill 毎ターン処理後 | 要求枚数を出し切れなかった側が敗北 |

- 上記以外のタイミングでデッキが 0 枚になること自体は即座に敗北にならない
- 勝敗決定時に画面中央に「YOU WIN」「YOU LOSE」「DRAW」の演出を表示し、ホームに戻るボタンが出現する
