# プロダクトドキュメント

## ゲーム概要

Unity 6 で開発するオンラインマルチプレイヤーカードゲーム。
UGS Multiplayer Services を使ったオンラインマッチメイキングに対応。

## 実装済み機能

- BGM / SE 再生（音量調整・永続化）
- カードのフレーバーテキスト読み上げ（カードをプレイ・召喚したとき、自分・相手どちらのカードでも、事前生成した VOICEVOX 音声を再生。フレーバー未設定・音声未生成のカードは無音。連続再生時は前の読み上げを止めず重ねて鳴らす（一時的に複数同時に流れてもよい）。読み上げる声はカードエディタの「読み上げ話者」でカードごとに指定でき、未指定カードは生成ツールの既定話者を使う）
- Commonシーンをベースとしたアディティブシーン管理
- フェードイン/アウトによる画面遷移演出
- オプションモーダル（BGM音量・SE音量・ボイス音量・Auto OK・タイトルに戻る。ボイス音量は読み上げ音声専用で SE とは独立。モーダルは開く＝Enter2 SE / 閉じる＝Cancel1 SE。ゲーム中は「タイトルに戻る」が「降参」になり、押すと確認ダイアログを開き（Enter SE）、「はい」＝Enter3 SE / 「いいえ」＝Cancel1 SE。チュートリアル中は勝敗のない練習のため「降参」ではなく「ホームに戻る」になり、確認後にホームへ戻る）
- ホームシーン Live2D キャラクター表示（HomeLive2DPresenter）：Dog-kid モデルを画面左側に配置。登録した全モーションからランダムに1つ選んで再生→終了後に次をランダム選択のループ。Animator + AnimationClip で再生し、CubismFadeController は起動時に無効化（FadeMotionList 未設定時の NullRef 回避）
- ホームシーン 犬（ペット）のセリフ（DogSpeechPresenter）：頭上の吹き出しに1文字ずつタイプ表示する（`DogSpeechLinesSO` に通常時・食事時・雨天時のセリフを登録し、状況に応じてランダム選択。`{name}` はユーザー名に置換して表示）
- ホームシーン クレジットモーダル：右下の「クレジット」ボタンで制作スタッフ・テスター・サウンド・使用OSS/アセット（Asset Store 含む）を記載したモーダルを開閉（HomePresenter で開閉。スクロール可能・「閉じる」ボタンで閉じる）
- ホームシーン 詳しいルール（遊び方）モーダル：右下「クレジット」の左にある「詳しいルール」ボタンで、新規ユーザー向けにゲームルールを解説したモーダルを開閉（HomePresenter で開閉）。目的／流れ／カード／バトル／オーバーリミット／能力／デッキの7タブで分類し、タブで表示を切り替える。デッキ切れ勝ち（空デッキからのドロー・ミルで敗北＝オーバーリミット）の仕組みは「オーバーリミット」タブで解説。守護・速攻・飛行・防人・強襲・デッキ攻撃×のキーワード能力も「能力」タブで解説。タブ切り替え時はスクロールを先頭へ戻す。各タブの内容は UXML に直書きし表示切替で出し分ける（追加アセット不要）
- チュートリアル（誘導つきスクリプト対戦）：Home 右下「チュートリアル」ボタンで選択モーダル（タブ：「基本・勝ち方」／「キーワード能力」）を開き、各項目を選ぶと固定デッキの1人用スクリプト対戦を開始する。`TutorialModel`（Common・消費型フラグ＋`TutorialId`）に種別をセットして Main へ遷移し、`MainPresenter` が `_isTutorial` で分岐。決定的セットアップ（先攻プレイヤー固定・固定デッキ・シャッフルなし・盤面プリセット）、VS告知/コイントス/マリガン省略・手番タイマー無効、台本どおりに動く CPU、コーチ吹き出し（タップで畳んで再表示できる・本文は左寄せ）で1アクションずつ誘導する。本番のゲームロジックは触らず `if (_isTutorial)` のフック＋コーチ層を被せるだけ（実装は [MainPresenter.Tutorial.cs](../Assets/Scripts/Main/MainPresenter.Tutorial.cs)）。内訳は「基本の遊び方」（出す→コスト→ターン終了でクリア）、「攻撃の仕方」（出す→召喚酔いで攻撃できないと知る→ターン終了→相手がタップした的を作る→次の番でタップした相手を攻撃。アンタップには攻撃できない／反撃ダメージなし）、勝ち方3種（デッキ切れ／制圧／勝利点。各 winReason 付きで「○○勝利」表示）、キーワード能力6種（守護=C1005／速攻=C1009／飛行=C5003／防人=C3007／強襲・デッキ攻撃×=C3006 の実在カードで体験。各エントリにキーワードアイコン表示）。基本・キーワードのクリアは「チュートリアル完了」オーバーレイ。誤った操作をすると「チュートリアル失敗」オーバーレイ（現状は飛行チュートリアルで守護持ちを攻撃したとき＝飛行は守護を無視して奥のキャラを狙うのが正解。守護持ちは攻撃対象として強調しないが攻撃自体は可能で、攻撃すると失敗）。クリア・失敗後のオーバーレイは再戦に意味がないため「ホームに戻る」ボタンのみ表示する（再戦ボタンは出さない。勝ち方3種も同様）。フラグは消費型のため中断しても通常 CPU 戦・オンライン戦に持ち越さない
- CPU 相手選択：Home「CPU対戦」を押すと相手選択カルーセル（格ゲ風・1体ずつ大きく表示し◀▶で送る／端で循環・ページドット表示）が開き、相手の名前とポートレートを確認して「この相手と戦う」で対戦開始。相手の名前・ポートレート・デッキ・難易度は `CpuRosterSO`（Addressables `Card/CpuRoster`）が相手ごとに保持し、再戦は同じ相手になる。難易度はカルーセルには表示せず（相手名に含める運用）、対戦中の CPU 挙動にのみ反映される。各相手のデッキは Editor の CPU デッキビルダー（相手スロット式）で編集する。ロスター未設定時は「CPU 1〜8」のプレースホルダーで既存 `CpuDeck.asset` を使う
- ホームシーン ユーザーネーム表示・編集：画面左上に現在のユーザーネームを表示し、同じ枠の右側に鉛筆アイコン（✎）の編集ボタンを一体化して配置。押すと変更モーダルが開き、現在名をプリフィルした入力欄で名前を編集できる。タイトルの初回入力と同じ `UsernameValidator`（空入力NG・16文字＝全角8文字制限）でリアルタイム検証し、無効な間は「決定」ボタンを無効化。決定で `UsernameRepository` に保存しラベルへ即反映（HomePresenter で開閉・右上「✕」で閉じる）
- Home の各モーダル（相手選択・チュートリアル・詳しいルール・クレジット・ユーザーネーム編集）は右上の丸い「✕」ボタンで閉じる（パネル最後の子に配置して最前面でクリックできるようにする）
- ホームシーン対戦ガード：デッキが 30 枚でないとき CPU対戦（相手選択を開く前）・オンライン対戦ボタンを押すとトーストを表示して遷移をブロック（不足時「デッキが30枚になっていません」・枚数超過時「デッキが30枚を超えています」）。コスト合計に上限はない。Editor 再生時のみ左上の「デッキ枚数制限」トグルを OFF にすると 30 枚ちょうどでなくても（1 枚以上なら）対戦を開始できる（空のときは「デッキが空です」）
- オンラインマッチング（クイックマッチ・ルーム一覧から手動参加（満員ルームは非表示）・クイックマッチ30秒/手動ルーム120秒タイムアウト。同時押しの競合は再照会＋IDタイブレークで解消）。背景画像（Addressables `Image/MatchingBackground`）をルート要素に scale-and-crop で表示
- 対戦開始待機表示（手札ドローアニメーション開始まで「対戦相手を待っています...」を全画面表示。オンライン対戦のデッキ交換待機中にも機能）
- カードゲームのゲームプレイロジック
  - 先攻・後攻決定（配牌前にコイントス演出で先攻・後攻を決定（「先攻！」/「後攻！」アナウンス。コイン回転中は Coin SE を鳴らし回転終了で停止、結果表示に合わせて Ready SE）。先攻有利は手札枚数差（先攻3枚・後攻5枚）と両プレイヤーの初手ドローなしで補正。なお対戦開始の VS 告知（○○ VS ××）では Battle SE を再生）
  - マリガン（先攻後攻決定・配牌の後に「マリガンしますか？」YES/NO でプレイヤーが任意選択。YES/NO クリックで Enter3 SE。確認中も手札カードをクリックしてカード詳細を閲覧できる。再ドローは初期手札枚数（先攻3/後攻5）。CPU は初期手札にキャラカードが1枚もない場合のみ自動マリガン。「マリガン」アナウンス演出あり）
  - ターン管理・フェーズ構成（ドローフェーズ → メインフェーズ（1アクション選択：キャラを出す / イベントを使う / 攻撃する / パス））
  - 手番の制限時間（自分のメインフェーズに90秒。左下のターン表示の上にカウントダウンを表示し、残り10秒以下で数字が赤く拡大＋減るたびに左右に揺れ＋毎秒の警告SE。0で「時間切れ！」告知→自動パス。途中でカードを出しかけ・コスト選択中でも手札へ戻して強制パス。カード効果の解決演出中は中断せず演出完了後にパス。タイマーは自分の手番のみ稼働し、CPU・オンライン相手の手番は各クライアントが管理）
  - 手札上限（手札は8枚まで。満杯時に手札へ入るカードはドロー・Draw効果・Bounce・Switch のいずれでも墓地へ送る＝バーン。自分のドロー時のみ「手札が上限 → 墓地へ」トースト表示。詳細は [docs/rules.md](rules.md)「手札の上限」）
  - メインフェーズ（キャラを出す）：手札キャラカードをフィールドに表向きで配置。ドロップ直後にコスト選択 UI 表示 → 手札からコスト枚数選択 → OK で確定（コストエフェクト演出と同じタイミングで CardUse SE。自分・相手とも）
  - メインフェーズ（イベントを使う）：手札イベントカードをフィールドへドロップ → コスト選択 → OK で即時解決（コストエフェクト演出と同じタイミングで CardUse SE。効果適用 → 墓地）
  - メインフェーズ（攻撃する）：フィールドキャラから矢印をドラッグして相手キャラへ攻撃（**攻撃対象はタップ状態のキャラのみ**）。ダメージ = ATK。ダメージ > 0 で相手キャラの HP を減少させ、HP ≤ 0 でキャラ破壊
  - タップ：攻撃したキャラは90°横向きにタップする。**タップ状態のキャラにしか攻撃できない**（未タップ＝未行動のキャラは狙えない）。自分のターン開始時に自分の場を全アンタップ、守護・防人は毎ターン終了時に自動でタップする（相手ターン中は常に攻撃対象になる）。デッキ攻撃はタップ制限の対象外だが攻撃側はタップする。判定は `CanAttackChar`（`CardView.IsTapped`）。カード効果の対象選択には影響しない
  - メインフェーズ（デッキへの直接攻撃／ミル）：フィールドキャラから矢印を相手デッキへドラッグすると、ATK 枚だけ相手デッキ上から墓地へ送る（デッキ枚数が ATK 未満なら全枚数）。デッキダメージで1枚墓地へ送られるたびに DeckDamage SE（デッキ攻撃・デッキダメージ効果の共通ミル処理。ダメージトリガー発動分は墓地送りでないため対象外）。ATK 0 はシールドアイコンで不発。攻撃制限・召喚酔いはキャラ攻撃と共通（そのキャラのターン攻撃を消費）。デッキを0にすると相手はデッキ切れで敗北。守護か防人がいると非飛行は狙えず、飛行は守護を貫通するが防人がいると狙えない（`CanAttackDeck`）。CPU は lethal なら最優先、盤面に攻撃対象がなければチップミルに使う
  - ATK = キャラATK
  - ATK=0 時はシールドアイコンをフローティング表示して攻撃演出をスキップ
  - イベントカード効果（手札のイベントカードをプレイして即時解決。Draw / Recover / BanishChar / Switch / Evolve / CostBoost / DamageAllEnemies / DamageEnemy / SummonChar / SummonFromDeckByKeyword / SummonFromGrave / CopyFieldChar / HealAllAllies / Bounce / BounceAll / DrawSkipNext / DrawNextTurnStart / AtkBoost / HpBoost / GrantKeyword / GainVPPerGreenGrave / NextCardCostFree / ExtraTurn / DamageEnemyDeck / DamageBothDecks などを実装。各効果の挙動・値の意味・対象選択・オンライン同期は [docs/event.md](event.md) を参照）
  - キャラ効果（キャラカードに EffectTrigger / EffectType / EffectValue を設定すると発動。発動タイミング（トリガー）は OnEnter（登場時）/ OnAttack（攻撃時）/ OnDestroy（破壊時）/ OnUsedAsCost（コストとして使用）/ OnTurnStart（自分のターン開始時）/ OnAttacked（被攻撃時）/ OnKill（撃破時）/ OnDealPlayerDamage（相手デッキにダメージを与えた時）。効果種別はイベントと共通の EventType を流用する。各トリガーの正確な発動条件・実装済み効果の一覧は [docs/event.md](event.md) を参照。固定値の勝利点付与は EventType ではなく全カード共通の VictoryPointBonus 付帯値で行う（効果と同時に発動））
  - コスト倍化（CostBoost）（キャラ: OnUsedAsCost + CostBoost、イベント: EventType=CostBoost。手札からコストとして支払うとき EffectValue/EventValue 分のコストとして数え、最小枚数で支払える。属性連動でこのカードの属性がプレイするカードの属性と一致するときのみ倍化、それ以外の属性のコストには1（白も一般属性扱いで、白 CostBoost は白のコストのみ倍化）。通常プレイ時は無効果。説明は Description に手書き）
  - キャラ破壊（HP ≤ 0 でパーティクル演出（Down SE）→カードをフィールドから除去→墓地へ飛翔。戦闘・効果・BanishChar すべての破壊経路で鳴る）
  - 共通の勝利条件（属性に依らずゲーム開始時から両プレイヤーに適用。`WinRule` / 勝因種別 `WinReason`。詳細は docs/rules.md「勝敗条件」）
    - デッキ切れ＝オーバーリミット（デッキが0枚の状態でカードを引く／ミルしようとした瞬間に本人が敗北＝相手が勝利。デッキを0枚にした引き／ミルそのものでは負けず、その後さらに引く／ミルする手番で敗北する。ターン最初のドローは例外で常に安全＝デッキ枚数分だけ引いて止まる。カードを引く／ミルする直前に `CheckDeckOutWin` で判定。一連の引き／ミルを生き残ってデッキが初めて0枚になったら「オーバーリミット！」を告知＝OverLimit SE。`Draw` 効果の値2=1 指定時は空デッキでも敗北せずオーバーリミット告知のみ）
    - 勝利点20（勝利点カウンター（メダル+数字。自分=左下/相手=右上）を開始時から両者に常時表示。`VictoryPointBonus` 付帯値（全属性で設定可）や `EventType.GainVPPerGreenGrave`（墓地の緑カード枚数で加点）で加点し、20到達で勝利。加点時は発動カード上にメダルフロート（得点数 1〜5 に応じた Medal1〜Medal5Icon。メダルアイコンが浮かび上がるタイミングで VictoryPoint SE）→カウンターで「+N」とカウントアップ演出。`VictoryPointsView`）
    - キャラ8体（自フィールドにキャラを8体同時に並べた側が勝利。キャラ配置・召喚の直後に `CheckFieldCharsWin` で判定。フィールド上限は9体）
  - 勝敗演出（共通の勝利条件で決着した場合は勝因テキスト（「デッキ切れ勝利」「勝利点勝利」「制圧勝利」など）を勝因色で表示し、対応アイコン（Grave/Medal/Attack）を紋章としてスケールイン。降参・タイムアウト・引き分けは YOU WIN 金 / YOU LOSE 赤 / DRAW 灰 をスケールアップで表示。いずれも「再戦する」「ホームに戻る」ボタンが出現（どちらもクリック時に Enter3 SE。ただしチュートリアルは再戦ボタンを出さず「ホームに戻る」のみ）。オーバーレイ登場に合わせて BGM を停止し、勝敗 SE を再生（勝利＝Win / 敗北＝Lose・引き分けは無音）。演出中も暗幕越しにフィールドの表向きキャラカードをクリックしてカード詳細を閲覧できる）
  - 再戦（対戦終了後に「再戦する」。オンラインは双方が押すと `NGS_Rematch` ハンドシェイクで Main シーンを再ロードして新規対戦を開始、CPU 戦は即再戦。待機中に相手が退出すると通知。詳細は docs/networking.md「再戦」）
  - CPU エージェント（各フェーズでカードを自動選択。難易度（初級/中級/上級）を相手ごとに設定でき、中級以上は CostBoost／ダメージトリガー持ちを場に出さずコスト支払いに優先的に回す）
  - イベント効果ラベル演出（Draw / Recover / BanishChar / Evolve）はフローティングラベルで表示（Switch はラベルなし＝パーティクルのみ）
  - 攻撃アニメーション（アタッカーが目標方向 60% へ突撃して戻るモーション。突撃開始に合わせて Attack SE。キャラ攻撃・デッキ攻撃の両方）
  - 攻撃ダメージ演出（ダメージ > 0 時、ヒットパーティクル → HP アイコン 2.2倍スケールアップ＋赤発光＋カード横揺れシェイク → アニメーション完了後に HP 数値を更新）
  - ターン開始告知演出（自分の番は「YOUR TURN」水色（告知に合わせて PlayerTurn SE）/ 相手の番は「ENEMY TURN」赤（SE なし）。ほかに「マリガン」「PASS」告知あり。ドロー/メインのフェーズ告知は廃止）
  - デッキ枚数表示をデッキスロット中央に配置（HeartIcon を背面に敷き、その上に枚数数字を重ねて表示。黒アウトライン付き）
  - PASS アニメーション（パス操作時に画面中央に「PASS」テキストをアニメーション表示。プレイヤー=水色 / CPU=赤）
  - ターン開始ドロー（ターン開始告知の完了後にデッキ上部から 3 枚引く。自分・CPU ともに毎ターン。ただし両プレイヤーの初手（各自がこのゲームで最初に迎える自分のドローフェーズ）はドローなし。枚数表示はドロー開始タイミングで更新。カードを1枚引くたびに Card SE を再生（ターンドロー・Draw 効果・配牌・マリガン再配牌のすべて、自他とも））
  - CPU ドロー演出（CPUのデッキから手札エリアへカードが飛翔するアニメーション）
  - 墓地アイコン（80×80px の GraveIcon + 枚数表示。クリックでカード一覧モーダルを開く。モーダルは横スクロール・トップ/ボトムラベル付き）
  - カード属性フレーム（Red/Blue/Green/Yellow/Black/Purple/White の属性色 2px カラーボーダー。裏向き時は非表示。カード詳細モーダルのイラスト枠も同じ属性色）
  - カード詳細モーダルの属性アイコン（タイプバッジ右に属性ごとのアイコン画像を表示。Red=Fire / Blue=Water / Green=Medal / Yellow=Dice / Black=Clock / Purple=Devil / White=Flower）
  - カード詳細モーダルの閉じ方（パネル右上の丸い✕ボタン、または外側の暗幕クリックで閉じる。閉じ判定は `ClickEvent` ではなく `PointerDownEvent`＝オンライン相手ターン中のフレームヒッチでも取りこぼさず確実に閉じられる）
