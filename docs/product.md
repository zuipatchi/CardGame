# プロダクトドキュメント

## ゲーム概要

Unity 6 で開発するオンラインマルチプレイヤーカードゲーム。
UGS Multiplayer Services を使ったオンラインマッチメイキングに対応。

## 実装済み機能

- BGM / SE 再生（音量調整・永続化）
- Commonシーンをベースとしたアディティブシーン管理
- フェードイン/アウトによる画面遷移演出
- オプションモーダル（BGM音量・SE音量・タイトルに戻る）
- ホームシーン Live2D キャラクター表示（HomeLive2DPresenter）：Dog-kid モデルを画面左側に配置。登録した全モーションからランダムに1つ選んで再生→終了後に次をランダム選択のループ。Animator + AnimationClip で再生し、CubismFadeController は起動時に無効化（FadeMotionList 未設定時の NullRef 回避）
- ホームシーン対戦ガード：デッキが 30 枚でないとき CPU対戦・オンライン対戦ボタンを押すとトーストを表示して遷移をブロック（不足時「デッキが30枚になっていません」・枚数超過時「デッキが30枚を超えています」）。コスト合計に上限はない
- オンラインマッチング（クイックマッチ・ルーム一覧から手動参加（満員ルームは非表示）・20秒タイムアウト）
- 対戦開始待機表示（手札ドローアニメーション開始まで「対戦相手を待っています...」を全画面表示。オンライン対戦のデッキ交換待機中にも機能）
- カードゲームのゲームプレイロジック
  - マリガン（ゲーム開始時に「マリガンしますか？」YES/NO でプレイヤーが任意選択。確認中も手札カードをクリックしてカード詳細を閲覧できる。CPU は初期手札にキャラカードが1枚もない場合のみ自動マリガン。「マリガン」アナウンス演出あり）
  - 先攻・後攻決定（マリガン後にコイントス演出で先攻・後攻を決定（「先攻！」/「後攻！」アナウンス）。補正ドローはなく、先攻有利は先攻の初手ドローなしで補正）
  - ターン管理・フェーズ構成（ドローフェーズ → メインフェーズ（1アクション選択：キャラを出す / イベントを使う / 攻撃する / パス））
  - メインフェーズ（キャラを出す）：手札キャラカードをフィールドに表向きで配置。ドロップ直後にコスト選択 UI 表示 → 手札からコスト枚数選択 → OK で確定
  - メインフェーズ（イベントを使う）：手札イベントカードをフィールドへドロップ → コスト選択 → OK で即時解決（効果適用 → 墓地）
  - メインフェーズ（攻撃する）：フィールドキャラから矢印をドラッグして相手キャラへ攻撃。ダメージ = ATK。ダメージ > 0 で相手キャラの HP を減少させ、HP ≤ 0 でキャラ破壊
  - ATK = キャラATK
  - ATK=0 時はシールドアイコンをフローティング表示して攻撃演出をスキップ
  - イベントカード効果（Draw: 発動プレイヤーが指定枚数ドロー。デッキが0枚になった時点で残りのドローを中断 / Recover: 自分の墓地の上から指定枚数を取り出して自デッキに加えてシャッフル。墓地枚数が不足する場合は全枚数を回収 / BanishChar: 相手フィールドの先頭キャラを墓地へ送る。相手フィールドにキャラがいない場合は空振り / Switch: 自分のフィールドのキャラカードを手札に戻した後、手札からキャラカードを1枚選んで表向きで配置してコストを払う。フィールドが空の場合は効果失敗。CPU は手札の最初のキャラを自動選択 / Evolve: 自分のフィールドのキャラを墓地に送り、手札から犠牲キャラより高コストのキャラを1枚コストなしで配置 / CostBoost: 通常プレイ時は無効果。手札からコストとして支払うときに EventValue 分のコストとして数える（コスト倍化）。属性連動でこのカードの属性がプレイするカードの属性と一致するときのみ倍化、それ以外は1（白も一般属性扱い） / DamageAllEnemies: 発動側から見た敵フィールドのキャラ全員に EventValue 分のダメージを同時に与え、HP 0 以下を破壊。敵フィールド中央に AoE パーティクル演出を再生し、敵キャラが0体でも演出は再生 / DamageEnemy: 発動側から見た敵キャラを値1体選び、それぞれに値2分のダメージを同時に与え、HP 0 以下を破壊。対象はプレイヤーがクリックで複数選択（選択済みは赤枠、残り体数をトースト表示）。対象数が敵の数以上なら全員・0体なら空振り。CPU は攻撃力上位を狙い、オンラインは対象をインデックス配列で同期 / SummonChar: 自フィールドに指定キャラ（値1=ID数字→"C###"）を値2体（0は1体）新規生成して配置。手札・デッキは消費せず、召喚キャラの OnEnter も発動。フィールド9体上限で打ち切り（連鎖も自然停止） / HealAllAllies: 発動側の自フィールド全キャラの HP を EventValue 分回復（値0で最大HPまで全回復）。自キャラが0体なら空振り / Bounce: 発動側から見た敵キャラを値1体選び、所有者（相手）の手札へ戻す。対象数が敵数以上なら全員・0体なら空振り。対象選択は DamageEnemy と同じ仕組みを共用。CPU は攻撃力上位を狙い、オンラインは対象をインデックス配列で同期）
  - キャラ効果（キャラカードに EffectTrigger / EffectType / EffectValue を設定すると発動。OnEnter（登場時）: 通常配置でフィールドに出した瞬間に発動。CPU・オンライン相手の配置でも発動。Switch / Evolve での配置は対象外。OnAttack（攻撃時）: 攻撃宣言時に発動。キャラ攻撃・ハート攻撃両方に対応。OnDestroy（破壊時）: 戦闘での撃破・DamageEnemy/DamageAllEnemies での撃破・BanishChar での除去で発動。破壊されたキャラを source として効果を解決する。OnUsedAsCost（コストとして使用）: EffectType=CostBoost と併用し、コスト支払い時に EffectValue 分のコストとして数える。実装済み効果: Draw（指定枚数ドロー）・BanishChar（相手フィールド先頭キャラを墓地へ）・DamageAllEnemies（敵キャラ全体にダメージ）・DamageEnemy（敵キャラを値1体選んで各値2ダメージ・対象はプレイヤー選択）・SummonChar（指定キャラをN体召喚）・HealAllAllies（自フィールド全キャラのHPを回復・値0で全回復）。固定値の勝利点付与は EventType ではなく全カード共通の VictoryPointBonus 付帯値で行う（効果と同時に発動）。OnTurnStart（自分のターン開始時）: 場にいる限り毎ターン1回発動。OnAttacked（被攻撃時）: 相手キャラの攻撃の対象になった直後に発動（攻撃側 ATK0 の盾ブロックでも発動。効果ダメージ・ハート攻撃では発動しない）。OnKill（撃破時）: 自分の攻撃で相手キャラを破壊したときに発動（戦闘のみ。効果破壊・ハート攻撃では発動しない））
  - コスト倍化（CostBoost）（キャラ: OnUsedAsCost + CostBoost、イベント: EventType=CostBoost。手札からコストとして支払うとき EffectValue/EventValue 分のコストとして数え、最小枚数で支払える。属性連動でこのカードの属性がプレイするカードの属性と一致するときのみ倍化、それ以外の属性のコストには1（白も一般属性扱いで、白 CostBoost は白のコストのみ倍化）。通常プレイ時は無効果。説明は Description に手書き）
  - キャラ破壊（HP ≤ 0 でパーティクル演出→カードをフィールドから除去→墓地へ飛翔）
  - ハート勝利条件（赤属性カードを1度でもプレイすると相手側にハート5個が永続出現。自分がプレイ→相手フィールド右上、相手がプレイ→自分フィールド左下。キャラから攻撃矢印 D&D でハートを攻撃でき、ATK 1 以上で1個破壊（縮小パーティクル演出付き）・ATK 0 はシールドアイコン表示。5個すべて破壊した側が勝利。CPU はハート攻撃可能なら最優先で狙う。詳細は docs/rules.md「勝敗条件」）
  - 青属性の勝利条件（青属性カードを1度でもプレイすると「自分のデッキが0枚になったら勝利」が武装され、自デッキ枚数表示に WaterIcon を重ねて表示。武装済みでデッキが0になった瞬間に勝利。未武装でデッキ0になっても無害。`BlueRule` / `DeckView.ShowBlueWinIcon`。詳細は docs/rules.md「勝敗条件」）
  - 緑属性の勝利条件（緑属性カードをプレイすると、プレイした側に勝利点表示（MedalIcon+数字。自分=左下/相手=右上）が出現。`VictoryPointBonus` 付帯値（効果とは独立した加点フィールド）や `EventType.GainVPPerGreenGrave` で加点し、20到達でそのプレイヤーが勝利。加点時は発動カード上に MedalIcon フロート→カウンターで「+N」とカウントアップ演出。`GreenRule` / `VictoryPointsView`。詳細は docs/rules.md「勝敗条件」）
  - 勝敗演出（色による勝利条件で決着した場合は「赤色勝利」「青色敗北」など勝因の色名テキストを属性色で表示し、属性アイコンを紋章としてスケールイン。降参・タイムアウト・引き分けは YOU WIN 金 / YOU LOSE 赤 / DRAW 灰 をスケールアップで表示。いずれも「再戦する」「ホームに戻る」ボタンが出現）
  - 再戦（対戦終了後に「再戦する」。オンラインは双方が押すと `NGS_Rematch` ハンドシェイクで Main シーンを再ロードして新規対戦を開始、CPU 戦は即再戦。待機中に相手が退出すると通知。詳細は docs/networking.md「再戦」）
  - CPU エージェント（各フェーズで最適カードを自動選択）
  - イベント効果ラベル演出（Draw / Recover / BanishChar / Switch / Evolve）はフローティングラベルで表示
  - 攻撃アニメーション（アタッカーが目標方向 60% へ突撃して戻るモーション）
  - 攻撃ダメージ演出（ダメージ > 0 時、ヒットパーティクル → HP アイコン 2.2倍スケールアップ＋赤発光＋カード横揺れシェイク → アニメーション完了後に HP 数値を更新）
  - ターン開始告知演出（自分の番は「YOUR TURN」水色 / 相手の番は「ENEMY TURN」赤。ほかに「マリガン」「PASS」告知あり。ドロー/メインのフェーズ告知は廃止）
  - デッキ枚数表示をデッキスロット中央に配置（枚数数字のみをカード中央に重ねて表示。黒アウトライン付き）
  - PASS アニメーション（パス操作時に画面中央に「PASS」テキストをアニメーション表示。プレイヤー=水色 / CPU=赤）
  - ターン開始ドロー（ターン開始告知の完了後にデッキ上部から 3 枚引く。自分・CPU ともに毎ターン。ただし先攻プレイヤーの初手（通算ターン1）はドローなし。枚数表示はドロー開始タイミングで更新）
  - CPU ドロー演出（CPUのデッキから手札エリアへカードが飛翔するアニメーション）
  - 墓地アイコン（80×80px の GraveIcon + 枚数表示。クリックでカード一覧モーダルを開く。モーダルは横スクロール・トップ/ボトムラベル付き）
  - カード属性フレーム（Red/Blue/Green/Yellow/Black/Purple/White の属性色 2px カラーボーダー。裏向き時は非表示。カード詳細モーダルのイラスト枠も同じ属性色）
  - カード詳細モーダルの属性アイコン（タイプバッジ右に属性ごとのアイコン画像を表示。Red=Fire / Blue=Water / Green=Medal / Yellow=Dice / Black=Clock / Purple=Devil / White=Flower）
  - フィールドのキャラ上限9体（`FieldView.MaxCharacters`。通常配置は満杯なら「フィールドがいっぱいです」トーストで拒否、CPU は満杯ならキャラを出さず攻撃/イベント/パスへ、召喚は満杯で打ち切り。イベントカードは墓地行きのため上限の対象外。6体以上は自動でスケール縮小。scale = max(0.4, 0.9 × 5/N)。カード間のマージンも補正して視覚的な間隔を維持）
  - 守護（タウント）（`CharacterCardData.Guardian`。守護持ちキャラが場にいる間、相手は守護にしか攻撃できずハートも狙えない。守護持ちはカード HP の下と詳細モーダルに ShieldIcon を表示。攻撃のみを制限しカード効果の対象選択には影響しない）
  - 速攻（`CharacterCardData.Haste`。速攻持ちキャラは召喚酔いせず、場に出したターンから攻撃できる。全配置経路で有効・1ターン1回の攻撃制限は維持。速攻持ちはカードと詳細モーダルに SpeedIcon を表示）
  - 飛行（`CharacterCardData.Flying`。飛行持ちキャラは守護を無視して攻撃対象を選べ、飛行を持つキャラからしか攻撃されない。飛行同士は攻撃可能。攻撃のみに作用しカード効果の対象選択には影響しない。飛行持ちはカードと詳細モーダルに FlyIcon を表示）
  - ドラッグ制御（自分のカード配置ターンでない場合・OKクリック後のフラッシュ演出中はカードドラッグを無効化。ドラッグ無効時もカードタップによる詳細モーダルは動作）
  - OK/戻るボタン画像（絶対配置・画像ボタン。Pass/OK は右端固定、戻るは OK の左 108px に配置）

  - コストシステム（カードプレイ時にコスト分を手札から選んで墓地へ送る。判定は枚数ではなく各カードの `CostPaymentValue` 合計。通常カードは1枚=1コスト、CostBoost カードは設定値分（ただし属性連動：プレイするカードの属性と一致するときのみ倍化、それ以外は1。白も一般属性扱い）。コスト選択後の OK ボタンで確定。プレイカードを除く手札の支払い可能量がコストに満たない場合はドロップを拒否）
  - コスト選択 UI ラベル（OKボタン上部に「コストを N 支払ってください」を常時表示。手札が不足の場合は「手札が足りません（N/cost）」、属性制約を満たさない場合は「「XXX」のカードを1枚以上含めてください」を表示し OK が無効化される）
  - デッキ構築（Home → DeckBuilder → Home の遷移。カード一覧をキャラ・イベントのセクション別に表示。カードをドロップしてデッキに追加（同一カードは3枚まで。4枚目を追加しようとすると「同名カードは3枚までです」トーストを表示して拒否）。カード一覧はカード全体を縮小表示。背景画像（Addressables）表示。ゴミ箱アイコンボタンでデッキを一括削除・デッキが空のときは非表示。フィルタボタン（キャラ・イベント種別）の右に7属性（赤/青/緑/黄/黒/紫/白）のアイコンボタンを配置。各ボタンは独立した ON/OFF トグルで、ON になっているカテゴリのカードのみ表示する（種別ボタンと属性ボタンの AND。例：キャラ ON・赤 ON のみなら赤のキャラのみ表示）。デッキ構築シーンに遷移したときは全ボタン ON（全表示）で始まる。ON のボタンはハイライト表示。全属性を OFF にすると何も表示されない。表示0枚になったセクションは見出しごと非表示。フィルタはカード一覧・デッキ一覧の両方に作用。フィルタ適用中もデッキの並べ替えが可能で、非表示カードの位置は保持される。その右に「枚数 N/30」を常時表示（コスト表示・コスト上限はなし）。30枚を超えると「30枚を超えています」の赤字ラベルをデッキタイトル上部に表示（position: absolute でレイアウト非影響）。デッキ一覧の各行は「名前 ／ 【－】 ／ 枚数 ／ 【＋】」で1枚単位で追加・削除可能。デッキ一覧のカード名・イラストをクリックでカード詳細モーダル表示。プレイヤー用・CPU用ともに DeckBuilderPresenterBase を継承した共通実装。同名3枚制限は Common 常駐の `DeckRuleModel` で保持し、Editor 再生時のみ Home シーン左上のトグルで ON/OFF 切り替え可能（ビルドでは常に ON。CPU デッキ構築は常に ON）。Addressables アセットのロード完了まで「読み込み中...」オーバーレイを全画面表示）
  - デッキ保存（PlayerPrefs に JSON 保存。カード追加・削除・一括削除のたびに自動保存。次回デッキ構築シーン起動時に復元）
  - デッキ分析モーダル（「分析」ボタンで開く。コスト別枚数分布とキャラ・イベント別枚数分布を棒グラフで表示。オーバーレイクリックまたは×ボタンで閉じる）
  - カード属性（Red / Blue / Green / Yellow / Black / Purple / White の7種。キャラ・イベント両カード対応。`CardAttribute` enum で定義。コスト支払い時の属性制約あり：白を含むすべての属性で、コストとして支払う手札に同属性のカードを 1 枚以上含める必要がある。白は万能コストではなく、白カードは他属性の属性要件を満たさない（コスト枚数の数合わせとしては選べる）。白カードをプレイする場合は白カードを最低1枚含める必要がある）

## 未実装（今後の課題）

- プレイヤープロフィール・ランキング
