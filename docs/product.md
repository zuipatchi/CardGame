# プロダクトドキュメント

## ゲーム概要

Unity 6 で開発するオンラインマルチプレイヤーカードゲーム。
UGS Multiplayer Services を使ったオンラインマッチメイキングに対応。

## 実装済み機能

- BGM / SE 再生（音量調整・永続化）
- Commonシーンをベースとしたアディティブシーン管理
- フェードイン/アウトによる画面遷移演出
- オプションモーダル（BGM音量・SE音量・タイトルに戻る）
- ホームシーン Live2D キャラクター表示（HomeLive2DPresenter）：Dog-kid モデルを画面左側に配置。登録した全モーションからランダムに1つ選んで再生→終了後に次をランダム選択のループ。CubismMotionController + CubismFadeController（フェード時間 0）使用
- ホームシーン対戦ガード：デッキが 30 枚でない・またはコスト合計が 80 を超えるとき CPU対戦・オンライン対戦ボタンを押すとトーストを表示して遷移をブロック（不足時「デッキが30枚になっていません」・枚数超過時「デッキが30枚を超えています」・コスト超過時「デッキのコストが80を超えています」）
- オンラインマッチング（クイックマッチ・ルーム一覧から手動参加（満員ルームは非表示）・20秒タイムアウト）
- 対戦開始待機表示（手札ドローアニメーション開始まで「対戦相手を待っています...」を全画面表示。オンライン対戦のデッキ交換待機中にも機能）
- カードゲームのゲームプレイロジック
  - マリガン（ゲーム開始時に「マリガンしますか？」YES/NO でプレイヤーが任意選択。CPU は初期手札にキャラカードが1枚もない場合のみ自動マリガン。「マリガン」アナウンス演出あり）
  - 先攻・後攻決定（マリガン後にコイントス演出で先攻・後攻を決定（「先攻！」/「後攻！」アナウンス）。後攻プレイヤーが1枚余分にドロー）
  - ターン管理・フェーズ構成（ドローフェーズ → メインフェーズ（1アクション選択：キャラを出す / イベントを使う / 攻撃する / パス））
  - メインフェーズ（キャラを出す）：手札キャラカードをフィールドに表向きで配置。ドロップ直後にコスト選択 UI 表示 → 手札からコスト枚数選択 → OK で確定
  - メインフェーズ（イベントを使う）：手札イベントカードをフィールドへドロップ → コスト選択 → OK で即時解決（効果適用 → 墓地）
  - メインフェーズ（攻撃する）：フィールドキャラから矢印をドラッグして相手キャラへ攻撃。ダメージ = ATK。ダメージ > 0 で相手キャラの HP を減少させ、HP ≤ 0 でキャラ破壊
  - ATK = キャラATK
  - ATK=0 時は「NO DAMAGE」ラベルを表示して攻撃演出をスキップ
  - イベントカード効果（Draw: 発動プレイヤーが指定枚数ドロー。次の1枚を引こうとした時点でデッキが0枚ならゲームオーバー / Recover: 自分の墓地の上から指定枚数を取り出して自デッキに加えてシャッフル。墓地枚数が不足する場合は全枚数を回収 / Switch: 自分のフィールドのキャラカードを手札に戻した後、手札からキャラカードを1枚選んで表向きで配置してコストを払う。フィールドが空の場合は効果失敗。CPU は手札の最初のキャラを自動選択）
  - キャラ破壊（ダメージ ≥ キャラの HP ならパーティクル演出→カードをスロットから除去→墓地へ飛翔。プレイヤー用・相手用で反転プレハブを使い分け）
  - 敗北条件（ドローフェーズ開始時・Draw エフェクト時にデッキが0枚なら敗北。コスト支払い時は払うコストがデッキ残枚数より多い場合に演出後に敗北（コスト == デッキ残枚数は続行）。デッキが0枚になること自体は即死しない）
  - 勝敗演出（YOU WIN 金 / YOU LOSE 赤 / DRAW 灰 をスケールアップで表示→ホームに戻るボタンが出現）
  - CPU エージェント（各フェーズで最適カードを自動選択）
  - イベント効果ラベル演出（Draw / Recover / BanishChar / Switch / Evolve）はフローティングラベルで表示
  - 攻撃アニメーション（アタッカーが目標方向 60% へ突撃して戻るモーション）
  - 攻撃ダメージ演出（ダメージ > 0 時、パーティクル → フローティング「-N」数字 → HP アイコンスケールアップ＋赤→白カラーアニメーションで HP 減少を演出）
  - フェーズ告知演出（「ドローフェーズ」水色系 / 「メインフェーズ」 / 「マリガン」 / 「PASS」プレイヤー=水色・CPU=赤）
  - フェーズリストパネル（画面左上に常時表示。ドロー→メインの2フェーズをドット＋縦棒線で接続し、現在フェーズのみハイライト）
  - デッキ枚数（ライフ）表示をデッキスロット中央に配置（ハートアイコン＋枚数数字がカード中央に重ねて表示）
  - PASS アニメーション（パス操作時に画面中央に「PASS」テキストをアニメーション表示。プレイヤー=水色 / CPU=赤）
  - ターン開始ドロー（告知アニメーション完了後にデッキ上部から 1 枚引く。自分・CPU ともに毎ターン。枚数表示はドロー開始タイミングで更新）
  - CPU ドロー演出（CPUのデッキから手札エリアへカードが飛翔するアニメーション）
  - 墓地アイコン（80×80px の GraveIcon + 枚数表示。クリックでカード一覧モーダルを開く。モーダルは横スクロール・トップ/ボトムラベル付き）
  - カード種別フレーム（キャラ=青・イベント=黄の 2px カラーボーダー。裏向き時は非表示）
  - カード種別ミスドロップ時トースト通知（キャラスロットにイベントカードをドロップ→「キャラカードのみセットできます」/ イベントフェーズのフィールドにキャラカードをドロップ→「イベントカードのみ使えます」。フェードイン→ホールド→フェードアウトで約1.5秒後に自動消滅）
  - フィールドカード枚数上限なし（6枚以上は自動でスケール縮小。scale = max(0.4, 0.9 × 5/N)。カード間のマージンも補正して視覚的な間隔を維持）
  - ドラッグ制御（自分のカード配置ターンでない場合・OKクリック後のフラッシュ演出中はカードドラッグを無効化。ドラッグ無効時もカードタップによる詳細モーダルは動作）
  - 手札ハイライト（自分の手番で出せるカードだけが境界線パルスで光る。キャラセットはキャラカード・イベントフェーズはイベントカードが対象。カードを置く・パスで消灯、戻るボタンで手札に戻ったカードは再点灯）
  - キャラセット配置時即裏向き（ドロップ直後にフリップアニメーション。戻るボタンで表向きに戻して手札へ返却）
  - OK/戻るボタン画像（絶対配置・画像ボタン。Pass/OK は右端固定、戻るは OK の左 108px に配置）

  - コストシステム（カードプレイ時にコスト枚数分を手札から選んで墓地へ送る。コスト選択後の OK ボタンで確定。払うコストが手札の残枚数より多い場合はドロップを拒否）
  - コスト警告（カードをフィールドにステージングした時点でコスト支払い後にデッキが0枚になる場合、OKボタン上部に赤字で警告メッセージを表示。Backで取り消し可能）
  - デッキ構築（Home → DeckBuilder → Home の遷移。カード一覧をキャラ・イベントのセクション別に表示。カードをドロップしてデッキに追加（枚数無制限に追加可能）。カード一覧はカード全体を縮小表示。背景画像（Addressables）表示。ゴミ箱アイコンボタンでデッキを一括削除・デッキが空のときは非表示。フィルタボタン（キャラ・イベント）の右に「枚数 N/30」「コスト N/80」を常時表示。30枚を超えると「30枚を超えています」・コスト合計が80を超えると「コストが80を超えています」の赤字ラベルをデッキタイトル上部に表示（position: absolute でレイアウト非影響）。デッキ一覧の各行は「名前 ／ 【－】 ／ 枚数 ／ 【＋】」で1枚単位で追加・削除可能。デッキ一覧のカード名・イラストをクリックでカード詳細モーダル表示。プレイヤー用・CPU用ともに DeckBuilderPresenterBase を継承した共通実装。Addressables アセットのロード完了まで「読み込み中...」オーバーレイを全画面表示）
  - デッキ保存（PlayerPrefs に JSON 保存。カード追加・削除・一括削除のたびに自動保存。次回デッキ構築シーン起動時に復元）
  - デッキ分析モーダル（「分析」ボタンで開く。コスト別枚数分布とキャラ・イベント別枚数分布を棒グラフで表示。オーバーレイクリックまたは×ボタンで閉じる）
  - カード属性（Red / Blue / Green / Yellow / Black / Purple / White の7種。キャラ・イベント両カード対応。`CardAttribute` enum で定義。コスト支払い時の属性制約あり：非 White のカードをプレイする場合、コストとして支払う手札に同属性または White のカードを 1 枚以上含める必要がある。White のカードをプレイする場合は任意属性のカードでコストを払える）
