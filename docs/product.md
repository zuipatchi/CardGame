# プロダクトドキュメント

## ゲーム概要

Unity 6 で開発するオンラインマルチプレイヤーカードゲーム。
UGS Multiplayer Services を使ったオンラインマッチメイキングに対応。

## 実装済み機能

- BGM / SE 再生（音量調整・永続化）
- Commonシーンをベースとしたアディティブシーン管理
- フェードイン/アウトによる画面遷移演出
- オプションモーダル（BGM音量・SE音量・タイトルに戻る）
- ホームシーン Live2D キャラクター表示（HomeLive2DPresenter）：Dog-kid モデルを画面左側に配置。登録した全モーションからランダムに1つ選んで再生→終了後に次をランダム選択のループ。CubismMotionController + CubismFadeController（フェード時間 0）使用
- オンラインマッチング（クイックマッチ・ルーム一覧から手動参加・20秒タイムアウト）
- カードゲームのゲームプレイロジック
  - マリガン（初期手札にキャラカードが1枚もない場合、自動的に手札を返却・リシャッフル・再ドロー。プレイヤー・CPU それぞれ1回限り。「MULLIGAN」アナウンス演出あり）
  - キャラセットフェーズ（ゲーム開始時1回のみ・先攻から順に裏向き配置・Resolve で同時表向き）
  - ターン管理・フェーズ構成（Draw → 戦闘前1 → 戦闘前2 → 解決 → 戦闘）
  - 戦闘前1フェーズ（キャラまたは技カードを1枚だけ裏向きでフィールドにセット）
  - 戦闘前2フェーズ（イベントカードのみ Ready 化。2連続パスで終了）
  - 解決フェーズ（Ready カードを逆順で処理・AtkBoost / DefBoost 効果を適用）
  - 戦闘フェーズ（全フィールドカードを表向き→フィールドのキャラはスロットへ移動→技カードはキャラスロットへ飛翔しキャラの下に重なる→ATKカウントアップ表示→技カードは墓地へ→スロットのキャラが突撃・戻る→デッキダメージ）
  - ATK は戦闘前1で出した技カードの Damage 合計（キャラスロットにキャラなし=0。戦闘前1でキャラを出したターンは攻撃しない）
  - イベントカード効果（AtkBoost: 解決フェーズで次の戦闘フェーズの ATK を加算 / DefBoost: 次の戦闘フェーズの有効 DEF を加算 / Draw: 発動プレイヤーが指定枚数ドロー。次の1枚を引こうとした時点でデッキが0枚ならゲームオーバー）
  - デッキ攻撃（(ATK + AtkBoost) - (DEF + DefBoost) の枚数分だけ相手デッキを墓地へ）
  - 敗北条件（ドローフェーズ開始時・コスト支払い時・Draw エフェクト時にデッキが0枚なら敗北。デッキが0枚になること自体は即死しない）
  - 勝敗演出（YOU WIN 金 / YOU LOSE 赤 / DRAW 灰 をスケールアップで表示→タイトルに戻るボタンが出現）
  - CPU エージェント（各フェーズで最適カードを自動選択）
  - Resolve テキスト演出（解決フェーズ開始時に画面中央に表示）
  - 攻撃アニメーション（アタッカーが目標方向 60% へ突撃して戻るモーション）
  - ダメージカード墓地送りアニメーション（ダメージ枚数分のカードがデッキから墓地アイコンへ表向きで飛翔・縮小しながら吸い込まれる。両者並行。デッキ枚数表示はカード1枚到着ごとに1ずつ減少）
  - ダメージ計算式表示（ATKカウントアップ後、DEF>0 のとき「ATK - DEF = ダメージ」を表示）
  - ターン告知演出（Resolve 完了後・ゲーム開始時に「YOUR TURN」水色 / 「ENEMY TURN」赤を表示）
  - フェーズ告知演出（"SET CHARACTERS" 緑 / "SET CARDS" 緑 / "SET EVENTS" 紫 / "FIGHT" 赤橙）
  - PASS アニメーション（パス操作時に画面中央に「PASS」テキストをアニメーション表示。プレイヤー=水色 / CPU=赤。キャラセット・戦闘前1・戦闘前2フェーズ対応）
  - ターン開始ドロー（告知アニメーション完了後にデッキ上部から 1 枚引く。自分・CPU ともに毎ターン。枚数表示はドロー開始タイミングで更新）
  - CPU ドロー演出（CPUのデッキから手札エリアへカードが飛翔するアニメーション）
  - 墓地アイコン（80×80px の GraveIcon + 枚数表示。クリックでカード一覧モーダルを開く。モーダルは横スクロール・トップ/ボトムラベル付き）
  - カード種別フレーム（キャラ=青・技=赤・イベント=黄の 2px カラーボーダー。裏向き時は非表示）
  - キャラセット配置時即裏向き（ドロップ直後にフリップアニメーション。戻るボタンで表向きに戻して手札へ返却）
  - OK/戻るボタン画像（絶対配置・画像ボタン。Pass/OK は右端固定、戻るは OK の左 108px に配置）

  - コストシステム（カードプレイ時にコスト枚数分を自デッキ上から墓地へ送る。キャラ・技カードはオープン時、イベントカードはReady確定時。コスト払いでデッキが0枚になった場合も負け判定）
  - コスト警告（カードをフィールドにステージングした時点でコスト支払い後にデッキが0枚になる場合、OKボタン上部に赤字で警告メッセージを表示。Backで取り消し可能）
  - チェーン番号表示（戦闘前2フェーズでReadyにしたカードに①②③…を中央に大きく表示。解決フェーズでそのカードが処理されるタイミングで消える）
  - デッキ構築（Title → DeckBuilder → Main の遷移。カード一覧をキャラ・技・イベントのセクション別に表示。カードをドロップしてデッキに追加。コスト合計がぴったり 30 になると「スタート」が有効になる（枚数制限なし）。カード一覧はカード全体を縮小表示。背景画像（Addressables）表示。ゴミ箱アイコンボタンでデッキを一括削除・デッキが空のときは非表示。デッキパネルヘッダー右にコスト（X/30）と枚数（X枚）を横並び表示。コストオーバー時に赤字ラベルをデッキタイトル上部に表示（position: absolute でレイアウト非影響）。デッキ一覧の各行は「名前 ／ 【－】 ／ 枚数 ／ 【＋】」で1枚単位で追加・削除可能。デッキ一覧のカード名・イラストをクリックでカード詳細モーダル表示。プレイヤー用・CPU用ともに DeckBuilderPresenterBase を継承した共通実装）
  - デッキ保存（PlayerPrefs に JSON 保存。カード追加・削除・一括削除のたびに自動保存。次回デッキ構築シーン起動時に復元）
  - デッキ分析モーダル（「分析」ボタンで開く。コスト別枚数分布とキャラ・技・イベント別枚数分布を棒グラフで表示。オーバーレイクリックまたは×ボタンで閉じる）
  - カード属性（Fire / Poison / Patchi の3種。キャラ・技カード両対応。AttributeIconDatabaseSO で属性→アイコンを一元管理。カード上にコスト・属性・ステータスの順で縦表示。デッキ構築画面のカード一覧にも表示）
