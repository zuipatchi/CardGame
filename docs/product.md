# プロダクトドキュメント

## ゲーム概要

Unity 6 で開発するオンラインマルチプレイヤーカードゲーム。
UGS Multiplayer Services を使ったオンラインマッチメイキングに対応。

## 実装済み機能

- BGM / SE 再生（音量調整・永続化）
- Commonシーンをベースとしたアディティブシーン管理
- フェードイン/アウトによる画面遷移演出
- オプションモーダル（BGM音量・SE音量・タイトルに戻る）
- ホームシーン Live2D キャラクター表示（HomeLive2DPresenter）：Dog-kid モデルを画面左側に配置。登録した全モーションからランダムに1つ選んで再生→終了後に次をランダム選択のループ。CubismMotionController + CubismFadeController（フェード時間 0）使用
- ホームシーン対戦ガード：デッキが 30 枚でない・またはコスト合計が 70 を超えるとき CPU対戦・オンライン対戦ボタンを押すとトーストを表示して遷移をブロック（不足時「デッキが30枚になっていません」・枚数超過時「デッキが30枚を超えています」・コスト超過時「デッキのコストが70を超えています」）
- オンラインマッチング（クイックマッチ・ルーム一覧から手動参加（満員ルームは非表示）・20秒タイムアウト）
- 対戦開始待機表示（手札ドローアニメーション開始まで「対戦相手を待っています...」を全画面表示。オンライン対戦のデッキ交換待機中にも機能）
- カードゲームのゲームプレイロジック
  - マリガン（初期手札にキャラカードが1枚もない場合、自動的に手札を返却・リシャッフル・再ドロー。プレイヤー・CPU それぞれ1回限り。「MULLIGAN」アナウンス演出あり）
  - 攻撃優先権システム（マリガン後にコイントス演出で優先権を決定。負けた側が1枚ドロー。素早さ同値のとき優先権保持者が先攻・行使後は相手に移る。保持者の墓地隣に72pxコインアイコン表示。行使時は「優先権 行使！」アナウンス＋コイン画面中央飛翔→墓地隣着地演出）
  - ターン管理・フェーズ構成（Draw → キャラセット → 戦闘前1 → 戦闘前2（解決を含む） → 戦闘）
  - キャラセットフェーズ（毎ターン Draw 直後に実行。両スロット埋まり時はスキップ。空スロットのプレイヤーがキャラカードを1枚裏向きで配置可（パス可）。Resolve 後に新規配置のみ表向き＋コスト払い）
  - 戦闘前1フェーズ（キャラまたはスキルカードを1枚だけ裏向きでフィールドにセット）
  - 戦闘前2フェーズ（イベントカードのみ Ready 化。2連続パスで終了→同フェーズ末に Ready カードを逆順で処理・AtkBoost / DefBoost 効果を適用）
  - 戦闘フェーズ（全フィールドカードを表向き→先攻コスト払い（ATK=0はスキップ）→先攻ATKカウントアップ→先攻突撃・ダメージ→後攻コスト払い（ATK=0はスキップ）→後攻ATKカウントアップ→後攻突撃・ダメージ→スキルカードは墓地へ）
  - ATK は戦闘前1で出したスキルカードの Damage 合計（キャラスロットにキャラなし=0。戦闘前1でキャラを出したターンは攻撃しない）
  - 先攻後攻決定：Speed 高い方が先攻。同値のとき攻撃優先権保持者が先攻（行使後は相手に移る）
  - キャラスロットにキャラカードがいない場合、有効DEF = 0（DefBoost も無効）
  - 先攻の攻撃でキャラが破壊された場合、後攻の ATK は 0 になる（反撃不可）
  - ATK=0 時は突撃せず「NO ATTACK」グレーラベルを表示（先攻・後攻ともに。先攻破壊による後攻ATK=0でも表示）
  - 戦闘ボーナスラベル演出（ATKカウントアップ後にATK数値の右側へスライドイン表示。スキルタイプ一致=オレンジ「スキルタイプ一致」×2 / 弱点ヒット=ゴールド「弱点を突いた」×3 / 得意属性耐性=グレー「効果がない」ATK=0。両者同時表示。タイプ一致＋弱点同時発生時は縦並びで2枚表示）
  - イベントカード効果（AtkBoost: 解決フェーズで次の戦闘フェーズの ATK を加算 / DefBoost: 次の戦闘フェーズの有効 DEF を加算 / Draw: 発動プレイヤーが指定枚数ドロー。次の1枚を引こうとした時点でデッキが0枚ならゲームオーバー / Recover: 自分の墓地の上から指定枚数を取り出して自デッキに加えてシャッフル。墓地枚数が不足する場合は全枚数を回収 / Switch: 自分のキャラスロットのカードを手札に戻した後、手札からキャラカードを1枚選んで表向きで配置してコストを払う。スロットが空の場合は効果失敗。CPU は手札の最初のキャラを自動選択）
  - デッキ攻撃（MAX(0, ATK + AtkBoost - DEF - DefBoost) の枚数分だけ相手デッキを墓地へ）
  - キャラ破壊（ダメージ ≥ キャラの HP ならパーティクル演出→カードをスロットから除去→墓地へ飛翔。プレイヤー用・相手用で反転プレハブを使い分け）
  - 敗北条件（ドローフェーズ開始時・Draw エフェクト時にデッキが0枚なら敗北。コスト支払い時は払うコストがデッキ残枚数より多い場合に演出後に敗北（コスト == デッキ残枚数は続行）。デッキが0枚になること自体は即死しない）
  - 勝敗演出（YOU WIN 金 / YOU LOSE 赤 / DRAW 灰 をスケールアップで表示→ホームに戻るボタンが出現）
  - CPU エージェント（各フェーズで最適カードを自動選択）
  - イベント効果ラベル演出（AtkBoost / DefBoost / Draw / Recover / Negate / BanishChar / Switch）はフローティングラベルで表示
  - Resolve テキスト演出（解決フェーズ開始時に画面中央に表示）
  - 攻撃アニメーション（アタッカーが目標方向 60% へ突撃して戻るモーション）
  - ダメージカード墓地送りアニメーション（ダメージ枚数分のカードがデッキから墓地アイコンへ表向きで飛翔・縮小しながら吸い込まれる。先攻→後攻の順に逐次実行。デッキ枚数表示はカード1枚到着ごとに1ずつ減少）
  - ダメージ数字飛翔演出（ATKカウンターアイコン突撃後、ダメージ > 0 の側はダメージアイコン＋ダメージ数字が相手キャラスロット中央でスケールイン出現→ホールド→相手デッキへ飛翔。相手スロット上に出現する場合は上下反転表示。先攻→後攻の順。飛翔完了後にデッキダメージ演出が始まり、デッキダメージ完了後にキャラ破壊演出＋墓地飛翔）
  - フェーズ告知演出（「ドローフェーズ」水色系 / 「キャラセットフェーズ」黒白縁取り / 「準備フェーズ」黒白縁取り / 「イベントフェーズ」黒白縁取り→「あなたが先です」水色or「相手が先です」赤 / 「FIGHT」黒白縁取り / 「PAY X COSTS」プレイヤー=水色・相手=赤）
  - フェーズリストパネル（画面左上に常時表示。ドロー→キャラ→準備→イベント→戦闘の5フェーズをドット＋縦棒線で接続し、現在フェーズのみハイライト。解決処理はイベントフェーズ末に自動実行されるため独立したフェーズ行なし）
  - デッキ枚数（ライフ）表示をデッキスロット中央に配置（ハートアイコン＋枚数数字がカード中央に重ねて表示）
  - PASS アニメーション（パス操作時に画面中央に「PASS」テキストをアニメーション表示。プレイヤー=水色 / CPU=赤。キャラセット・戦闘前1・戦闘前2フェーズ対応）
  - ターン開始ドロー（告知アニメーション完了後にデッキ上部から 1 枚引く。自分・CPU ともに毎ターン。枚数表示はドロー開始タイミングで更新）
  - CPU ドロー演出（CPUのデッキから手札エリアへカードが飛翔するアニメーション）
  - 墓地アイコン（80×80px の GraveIcon + 枚数表示。クリックでカード一覧モーダルを開く。モーダルは横スクロール・トップ/ボトムラベル付き）
  - カード種別フレーム（キャラ=青・スキル=赤・イベント=黄の 2px カラーボーダー。裏向き時は非表示）
  - カード種別ミスドロップ時トースト通知（キャラスロットにスキル/イベントカードをドロップ→「キャラカードのみセットできます」/ 準備フェーズのフィールドにイベントカードをドロップ→「スキルまたはキャラカードのみ使えます」/ イベントフェーズのフィールドにスキル/キャラカードをドロップ→「イベントカードのみ使えます」。フェードイン→ホールド→フェードアウトで約1.5秒後に自動消滅）
  - フィールドカード枚数上限なし（6枚以上は自動でスケール縮小。scale = max(0.4, 0.9 × 5/N)。カード間のマージンも補正して視覚的な間隔を維持）
  - ドラッグ制御（自分のカード配置ターンでない場合・OKクリック後のフラッシュ演出中はカードドラッグを無効化。ドラッグ無効時もカードタップによる詳細モーダルは動作）
  - 手札ハイライト（自分の手番で出せるカードだけが境界線パルスで光る。キャラセットはキャラカード・準備フェーズはスキル/キャラカード・イベントフェーズはイベントカードが対象。カードを置く・パスで消灯、戻るボタンで手札に戻ったカードは再点灯）
  - キャラセット配置時即裏向き（ドロップ直後にフリップアニメーション。戻るボタンで表向きに戻して手札へ返却）
  - OK/戻るボタン画像（絶対配置・画像ボタン。Pass/OK は右端固定、戻るは OK の左 108px に配置）

  - コストシステム（カードプレイ時にコスト枚数分を自デッキ上から墓地へ送る。キャラ・スキルカードはオープン時、イベントカードはReady確定時。払うコストがデッキ残枚数より多い場合は演出後に敗北。コスト == デッキ残枚数の場合は続行）
  - コスト警告（カードをフィールドにステージングした時点でコスト支払い後にデッキが0枚になる場合、OKボタン上部に赤字で警告メッセージを表示。Backで取り消し可能）
  - チェーン番号表示（戦闘前2フェーズでReadyにしたカードに①②③…を中央に大きく表示。解決フェーズでそのカードが処理されるタイミングで消える）
  - デッキ構築（Home → DeckBuilder → Home の遷移。カード一覧をキャラ・スキル・イベントのセクション別に表示。カードをドロップしてデッキに追加（枚数無制限に追加可能）。カード一覧はカード全体を縮小表示。背景画像（Addressables）表示。ゴミ箱アイコンボタンでデッキを一括削除・デッキが空のときは非表示。フィルタボタン（キャラ・スキル・イベント）の右に「枚数 N/30」「コスト N/70」を常時表示。30枚を超えると「30枚を超えています」・コスト合計が70を超えると「コストが70を超えています」の赤字ラベルをデッキタイトル上部に表示（position: absolute でレイアウト非影響）。デッキ一覧の各行は「名前 ／ 【－】 ／ 枚数 ／ 【＋】」で1枚単位で追加・削除可能。デッキ一覧のカード名・イラストをクリックでカード詳細モーダル表示。プレイヤー用・CPU用ともに DeckBuilderPresenterBase を継承した共通実装。Addressables アセットのロード完了まで「読み込み中...」オーバーレイを全画面表示）
  - デッキ保存（PlayerPrefs に JSON 保存。カード追加・削除・一括削除のたびに自動保存。次回デッキ構築シーン起動時に復元）
  - デッキ分析モーダル（「分析」ボタンで開く。コスト別枚数分布とキャラ・スキル・イベント別枚数分布を棒グラフで表示。オーバーレイクリックまたは×ボタンで閉じる）
  - カード属性（Fire / Poison / Patchi の3種。キャラ・スキルカード両対応。AttributeIconDatabaseSO で属性→アイコンを一元管理。カード上にコスト・属性・ステータスの順で縦表示。デッキ構築画面のカード一覧にも表示）
