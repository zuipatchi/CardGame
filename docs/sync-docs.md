## チェックリスト

### CLAUDE.md「主要ファイルの場所」テーブル

- [ ] 新しく追加したスクリプトが未記載になっていないか
- [ ] 削除・移動したスクリプトのパスが古いまま残っていないか
- [ ] コーディング規約の変更が反映されているか

### docs/architecture.md

- [ ] DI 登録（`*LifetimeScope.cs`）の変更が反映されているか
- [ ] シーン構成・遷移フローの変更が反映されているか
- [ ] `DeckModel`・`GameModel` など主要クラスの API 記述が実装と一致しているか
- [ ] Presenter のフロー記述が実装と一致しているか

### docs/rules.md

- [ ] ATK 計算式・倍率が `MainPresenter.Phases.cs` の実装と一致しているか
- [ ] ダメージ計算・コストシステムが実装と一致しているか
- [ ] フェーズフローが `TurnPhase` enum と `GameModel` の実装と一致しているか
- [ ] 勝敗条件が実装と一致しているか
- [ ] CPU デッキ仕様が `CpuDeckRepository` の実装と一致しているか

### docs/product.md

- [ ] 「実装済み機能」として記載されているものが実際にコードに存在するか
- [ ] デッキ構築ルール（コスト条件・枚数制限など）が実装と一致しているか

### docs/patterns.md

- [ ] 新しく追加したカード種別・フェーズ・EffectType の手順が記載されているか
- [ ] 既存パターンの手順が現在のコードと一致しているか

---

## 修正の判断基準

| 状況 | 対応 |
|---|---|
| コードが正しく、ドキュメントが旧仕様を指している | ドキュメントを修正する |
| ドキュメントが正しく、コードが未実装または誤実装 | 実装を修正するか、未実装機能として明記する |
| どちらが正しいか不明 | 担当者に確認してから判断する |

---

## よくある乖離パターン

- メソッド名・プロパティ名をリネームしたときに記述が古いまま残る
- `DeckModel` や `GameModel` に API を追加・削除したとき
- 計算式の定数（倍率など）を変更したとき
- フェーズの追加・変更・削除をしたとき
- スクリプトを新規作成または削除・移動したとき

---

## 自動チェック

`/sync-docs` スキルを使うと Explore エージェントが上記チェックリストを自動実行し、乖離箇所を検出・修正してコミットまで行う。
